# API Inquiry v0.2.0 Usage Console Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the v0.2.0 local usage console, DeepSeek Usage CSV import, local usage summaries, and move API key management from the menu bar panel into the console.

**Architecture:** `APIInquiryCore` gains pure Swift usage models, a CSV parser, a JSON store, and a console view model. `APIInquiryApp` gains a reusable console window and SwiftUI views. The menu bar continues sharing the balance controller but becomes a lightweight status and navigation surface instead of an API key form.

**Tech Stack:** Swift 5.9, SwiftUI, AppKit, Foundation CSV/JSON handling, macOS Keychain, Swift Package Manager, local `APIInquiryCoreTestsRunner`.
