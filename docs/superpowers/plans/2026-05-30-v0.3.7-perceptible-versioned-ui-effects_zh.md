# v0.3.7 可感知分版本 UI 效果实施计划

> **给 agentic workers：** REQUIRED SUB-SKILL: 使用 superpowers:subagent-driven-development（推荐）或 superpowers:executing-plans 按任务实施本计划。步骤使用 checkbox（`- [ ]`）语法跟踪。

**目标：** 在保持 macOS 13+ fallback 和 v0.3.7 克制 UI 方向的前提下，让更高版本 macOS SwiftUI API 在真实操作中更容易被感知。

**架构：** 继续把分版本 SwiftUI 调用集中在 `VersionedUIEffects.swift`。Core display model 暴露真实 numeric value，让 App 视图使用 `numericText(value:)`，避免解析格式化字符串。刷新可见性和结果反馈只做 App 层轻状态，不改变 provider、凭据或刷新架构。

**技术栈：** Swift 5.9、SwiftUI/AppKit、macOS 13+、Swift Package Manager、`APIInquiryCoreTestsRunner`、App target 构建验证。

---

## 范围边界

本轮包含：

- 为主金额和 quota 金额 display model 增加真实 numeric value，让 macOS 14+ `numericText(value:)` 真正生效。
- 为刷新符号增加确定性单圈反馈，让点击响应在快速刷新时也可见、只正向转动，并且持续时间与代码一致。
- 在 settings/result feedback 文案旁加入很小的状态图标，让保存/删除/错误反馈更容易扫到。
- 增加 numeric value 测试，并验证 macOS 13 fallback 仍然安全。

本轮不包含：

- 将最低系统版本提高到 macOS 14 或更高。
- Liquid Glass / `glassEffect`。
- Console 布局重设计、新卡片、新导航、dashboard 化、toast 系统或 modal feedback。
- Provider 扩展、动态 provider、凭据存储变化或 v0.4.0 工作。
- 解析格式化金额字符串来反推数值。
- 强动效、发光、弹跳、大面积颜色变化或装饰动画。
