# v0.3.7 UI 优化第一轮实施计划

> **给 agentic workers：** REQUIRED SUB-SKILL: 使用 superpowers:subagent-driven-development（推荐）或 superpowers:executing-plans 按任务实施本计划。步骤使用 checkbox（`- [ ]`）语法跟踪。

**目标：** 完成 v0.3.7 第一轮低风险 UI polish，不改变现有供应商工作流、布局识别度，也不进入 v0.4.0 provider 扩展范围。

**架构：** 只有当 App target 目前依赖本地化展示文案做判断时，才在 `APIInquiryCore` 补充小型语义状态。视觉 polish 留在 App target，通过现有 SwiftUI 视图和 `ConsoleDesignTokens` 完成。不引入新依赖、不做大视图拆分、不做 dashboard 化布局变化、不改 provider 模型。

**技术栈：** Swift 5.9、SwiftUI/AppKit、macOS 13+、Swift Package Manager、`APIInquiryCoreTestsRunner`。

---

## 范围边界

第一轮只包含刷新/错误/状态微交互、`APIAccessBadge` 语义状态修复、精简 help/accessibility 标签，以及在现有 token 层中轻量整理 tone color。

第一轮不包含 `MenuBarContentView` 拆分、Console Home 重设计、API 移除确认交互变化、新增供应商、动态 provider ID、v0.4.0 generic provider 能力，以及任何会抢夺额度/供应商/状态/凭据内容注意力的视觉处理。
