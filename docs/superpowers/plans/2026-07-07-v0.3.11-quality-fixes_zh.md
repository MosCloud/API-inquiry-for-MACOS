**目标：** 将 `v0.3.11` 做成聚焦的质量修复版本，解决 2026-07-07 质量评审中最高优先级的问题，但不启动 `v0.4.0` 的 Provider 泛化重构。

## 范围分类

### 纳入 `v0.3.11`

| 领域 | 问题 | 为什么属于本版本 |
| --- | --- | --- |
| CI | 没有 GitHub Actions 基线 | 收益高、代码风险低，可以防止无验证发布。 |
| Core 语义 | `"5h"` 和 `"Week"` 是逻辑判断键 | 成本低，可防止本地化或展示文案调整时静默破坏逻辑。 |
| Codex 手动重置 | 使用 `String.hashValue` 跟踪凭证变化 | 属于正确性风险，且可局部修复。 |
| Codex 凭证 | 异常 `auth.json` 被静默吞掉 | 最敏感 Provider 路径的用户诊断缺口。 |
| Console UI | 两条反馈通道可能叠显 | 用户可见噪音，ViewModel/UI 改动边界小。 |
| Console UI | 窗口尺寸有两个来源 | 用户可见布局漂移，适合用 token 收敛。 |
| 外观 | 关键 surface 仍使用深色模式白色描边 | 浅色模式真实可见问题。 |
| UI 重复 | 刷新反馈状态机重复 | 局部重构，减少真实重复，不改变功能。 |
| 发布文档 | 版本和 release docs | 干净关闭 `v0.3.11` 必需。 |

### 延后到 `v0.3.11` 之后

| 领域 | 延后项 | 目标 |
| --- | --- | --- |
