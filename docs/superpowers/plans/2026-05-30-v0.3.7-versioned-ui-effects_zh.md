# v0.3.7 分版本 UI 效果实施计划

> **给 agentic workers：** REQUIRED SUB-SKILL: 使用 superpowers:subagent-driven-development（推荐）或 superpowers:executing-plans 按任务实施本计划。步骤使用 checkbox（`- [ ]`）语法跟踪。

**目标：** 在保持 macOS 13+ fallback 的前提下，加入克制的高版本 SwiftUI 效果增强。

**架构：** 在 App target 中新增一个小型分版本效果 helper，集中管理 availability checks，避免散落在业务视图里。高版本 API 只用于提升状态、数字变化和结果反馈的清晰度。macOS 13 行为保持稳定，不做布局重设计。

**技术栈：** Swift 5.9、SwiftUI/AppKit、macOS 13+、Swift Package Manager、`APIInquiryCoreTestsRunner`、App target 构建验证。

---

## 范围边界

本轮包含：

- 为主金额和 quota 数字增加 numeric text transition。
- 为刷新按钮增加高版本 symbol effect，同时保留 macOS 13 fallback。
- 为设置/结果反馈增加高版本 sensory feedback，同时保留 macOS 13 fallback。
- 更新文档/进度，并进行 agent QA 审查。

本轮不包含：

- 将最低系统版本提高到 macOS 14 或更高。
- Liquid Glass / `glassEffect`。
- Console 布局重设计、新卡片、新导航或 dashboard 化。
- Provider 扩展、动态 provider、凭据存储变化或 v0.4.0 工作。
- 任何会分散额度、状态、供应商或凭据内容注意力的动画。

## 任务顺序

### Task 1：集中分版本效果 helper

**文件：**
- 创建：`Sources/APIInquiryApp/VersionedUIEffects.swift`

- [x] 增加 `apiInquiryNumericTextTransition(value:)`：macOS 14+ 且有真实数值时使用 `ContentTransition.numericText(value:)`，macOS 13 和 display-only 字符串 fallback 到 `ContentTransition.numericText(countsDown: false)`。
