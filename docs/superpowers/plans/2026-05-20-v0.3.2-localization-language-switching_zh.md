# API Inquiry v0.3.2 本地化与语言切换计划

> **给 agentic workers：** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**目标：** 为 API Inquiry 增加完整中文 UI 和 `Auto / 中文 / English` 语言切换能力，并保持现有 DeepSeek、智谱和 OpenAI/Codex 查询逻辑不变。

**架构：** 在 `APIInquiryCore` 增加轻量 localization layer，集中管理语言偏好、系统语言解析和 UI 文案；Core ViewModel/Formatter 输出已本地化字符串，SwiftUI 层只负责显示和少量控件绑定。语言偏好持久化到 `UserDefaults`，默认 `Auto`，切换后通过共享 observable state 触发菜单栏与 Console 更新。

**技术栈：** Swift 5.9、SwiftUI、AppKit、Combine、UserDefaults、Swift Package Manager、本地 `APIInquiryCoreTestsRunner`。

---

## 范围

### 包含

- 新增 `AppLanguage.auto / zh / en` 模型。
- 新增语言偏好 store，默认 `Auto`，并持久化用户选择。
- `Auto` 根据 macOS preferred languages 解析为中文或英文。
- 新增统一文案层，覆盖菜单栏详情、Console Home、Console API、按钮、状态、错误、设置反馈、更新时间和 reset 文案。
- Console 增加语言设置入口，优先放在 Home 顶部或轻量 Settings 区域。
- 手动切换语言后，菜单栏详情和 Console 尽量即时更新，无需重启。
- 更新 README、release notes 和 `Scripts/version.env` 到 v0.3.2。

### 不包含

- 不新增第三种语言。
- 不翻译 provider 品牌名。
- 不改变 provider 查询、credential、quota/balance parsing、历史数据结构。
- 不新增历史趋势图表或 CSV 导出。

## 开发流程

- [ ] 写失败测试：`AppLanguage(rawValue:)` 支持 `auto`、`zh`、`en`，默认值为 `.auto`。
