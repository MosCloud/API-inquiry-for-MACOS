# API Inquiry v0.3.1 Codex Quota Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a Codex provider that auto-reads local Codex login state, queries ChatGPT/Codex session quota, shows 5h and Week remaining quota, and displays the current plan in the Console.

**Architecture:** Add a `quotaUsage` branch to the existing `ProviderSnapshot` model. A Codex credential source first reads `$CODEX_HOME/auth.json` or `~/.codex/auth.json`, falling back to Keychain only when local Codex auth is unavailable. `CodexQuotaProvider` owns credential parsing and `/backend-api/wham/usage` response parsing, while formatters and ViewModels derive menu, detail, and Console strings so SwiftUI does not understand the network schema directly.

**Tech Stack:** Swift 5.9, SwiftUI, AppKit, URLSession-backed `HTTPClient`, macOS Keychain, UserDefaults, Swift Package Manager, local `APIInquiryCoreTestsRunner`.
