# v0.3.6 API Page UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Convert the Console API page into compact independent provider cards with clear access status, provider-specific purpose copy, and an OpenAI/Codex local config action.

**Architecture:** Add small display-model fields to `APIProviderSummary` so SwiftUI renders API-page status, purpose, and config actions without string inference. Keep the SwiftUI change local to `UsageConsoleView.swift`, reusing the existing Home provider chip helper. Extend `CodexCredentialStore` with a non-sensitive config target resolver used by the ViewModel.

**Tech Stack:** Swift 5.9 package, SwiftUI/AppKit on macOS, local executable test harness `APIInquiryCoreTestsRunner`.

---

## Task 2: Compact API Card SwiftUI

**Files:**
- Modify: `Sources/APIInquiryApp/UsageConsoleView.swift`

- [ ] **Step 1: Replace API panel layout with compact card**

Change `apiProviderPanel(_:)` so its default state is a compact card with:

```text
provider chip | apiAccessStatusText pill + apiAccessPurposeText | primary action + more menu
```

Reuse `providerHomepageButton(summary)` for the left chip. Do not create a second provider-logo style.

- [ ] **Step 2: Add primary action behavior**

For providers with `supportsAPIKeyManagement == true`:

- If unconfigured or replacing, show compact `SecureField` input state with save/cancel.
- Otherwise show `strings.replaceKey`.

For OpenAI/Codex:

- Show `strings.openConfig`.
- Open `summary.codexConfigTargetURL` through `NSWorkspace.shared.open`.
- If the URL is missing, show provider-scoped feedback with `strings.configCouldNotBeOpened`.

- [ ] **Step 3: Move destructive actions into a more menu**

Use a trailing menu button for:

- `strings.deleteKey` when key management is supported and configured.
- `strings.showConfigInFinder` for Codex when a config target exists.
- `strings.removeProvider`.

Keep existing deletion/removal confirmation state and disable removal when one provider remains.

- [ ] **Step 4: Keep confirmation and feedback compact**

Render provider removal confirmation, API key deletion confirmation, and feedback below the compact row inside the same card using small spacing and caption text.
