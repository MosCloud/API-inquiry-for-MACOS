# v0.3.9 Codex Manual Reset Credits Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Console Home-only Codex manual reset credit information without changing Menubar display or blocking the existing 5h/7d quota refresh path.

**Architecture:** Keep `/backend-api/wham/usage` as the main provider refresh path. Add a small reset-credits client and display/cache state owned by `UsageConsoleViewModel`, then expose a low-emphasis accessory refresh button only on the OpenAI Home metric.

**Tech Stack:** Swift 5.9 package, SwiftUI macOS app, custom executable test runner `APIInquiryCoreTestsRunner`, mock `HTTPClient` tests.

## Global Constraints

- Work on branch `v0.3.9`.
- Preserve the user-provided AGENTS.md behavior: think before coding, keep changes surgical, avoid speculative features, verify success.
- Menubar display must not show manual reset information.
- Menubar manual refresh must not query `/backend-api/wham/rate-limit-reset-credits`.
- Do not add reset credits to `BalanceRefreshController` or the provider main `ProviderSnapshot` refresh path.
- `Console Home` OpenAI card keeps `API 密钥`, `状态`, `详情`, and `更新于`.
- Replace OpenAI Home `计划` metric with `手动重置`.
- `详情` continues to show 5h remaining quota.
- Manual reset refresh is Console Home-only, local to the metric, non-blocking, and cached for 1 hour.
- The local icon-only refresh button uses `arrow.clockwise`, tooltip/accessibility `刷新手动重置信息`, and low-emphasis styling.
- Never print, persist, or expose access tokens, refresh tokens, cookies, full account IDs, Authorization headers, or raw reset credit payloads.
- Use TDD: add/adjust tests first, run them to observe failure, then implement minimal code.
