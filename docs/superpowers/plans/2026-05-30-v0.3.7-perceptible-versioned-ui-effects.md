# v0.3.7 Perceptible Versioned UI Effects Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make higher-version macOS SwiftUI APIs perceptible in real workflows while preserving macOS 13+ fallback and the restrained v0.3.7 UI direction.

**Architecture:** Keep version-specific SwiftUI calls centralized in `VersionedUIEffects.swift`. Expose raw numeric values from Core display models so App views can use `numericText(value:)` without parsing formatted strings. Use small App-only state for refresh visibility and lightweight result affordances without changing provider, credential, or refresh architecture.

**Tech Stack:** Swift 5.9, SwiftUI/AppKit, macOS 13+, Swift Package Manager, `APIInquiryCoreTestsRunner`, App target build verification.

---

## Scope Boundary

This pass includes:

- Real numeric values for primary amount and quota amount display models so macOS 14+ `numericText(value:)` actually runs.
- A deterministic one-turn refresh affordance so the click response is visible, forward-only, and duration-matched even when refresh completes quickly.
- A very small result feedback icon next to settings feedback text so save/delete/error feedback is easier to scan.
- Tests for numeric values and verification that macOS 13 fallback remains safe.

This pass excludes:

- Raising the deployment target above macOS 13.
- Liquid Glass / `glassEffect`.
- Console layout redesign, new cards, new navigation, dashboard behavior, toast systems, or modal feedback.
- Provider expansion, dynamic providers, credential storage changes, or v0.4.0 work.
- Parsing formatted amount strings to recover numbers.
- Strong motion, glow, bounce, large color washes, or decorative animation.
