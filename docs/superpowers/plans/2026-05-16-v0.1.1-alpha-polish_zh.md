# API Inquiry v0.1.1 Alpha 打磨实施计划

> **给 agentic workers：** 必须使用子技能：推荐使用 superpowers:subagent-driven-development，或使用 superpowers:executing-plans 按任务逐项执行本计划。步骤使用复选框 (`- [ ]`) 语法追踪。

**目标：** 将 v0.1.1 作为聚焦的 alpha 打磨版本交付，包含更清晰的首次设置、可操作的错误恢复、更安全的 API Key 管理反馈、开发过程态快速本地启动，以及更可信的 GitHub DMG 发布产物。

**架构：** 在 `APIInquiryCore` 中增加小型失败分类层，由 `MenuBarBalanceViewModel` 暴露派生 UI 动作，`MenuBarContentView` 保持声明式展示。开发验证保持本地优先，通过快速启动脚本验证；DMG 打包、checksum 生成和 release notes 检查只放在 release candidate 验证阶段。

**技术栈：** Swift 5.9、SwiftUI `MenuBarExtra`、macOS Keychain、Bash 打包脚本、ad-hoc `codesign`、`hdiutil`、基于 `shasum` 的 SHA-256 checksum。

---

本计划刻意保持 v0.1.1 小范围。不构建本地 DeepSeek 控制台、不做用量图表、不增加多供应商 UI、不做完整本地化、不引入 Developer ID 签名，也不做 Apple notarization 公证。

private static func failureKind(for error: Error) -> BalanceFailureKind {
    if let providerError = error as? BalanceProviderError {
        return providerError.failureKind
    }

    if let urlError = error as? URLError {
        switch urlError.code {
        case .notConnectedToInternet, .networkConnectionLost, .cannotFindHost, .cannotConnectToHost, .timedOut:
            return .networkUnavailable
        default:
            return .unknown
        }
    }

    return .unknown
}
```

INFO_PLIST="$VERIFY_MOUNT/$APP_NAME/Contents/Info.plist"
EXECUTABLE_PATH="$VERIFY_MOUNT/$APP_NAME/Contents/MacOS/APIInquiry"

if [ ! -x "$EXECUTABLE_PATH" ]; then
    echo "Executable missing from $DMG_NAME." >&2
    exit 1
fi

BUNDLE_ID=$(/usr/libexec/PlistBuddy -c "Print CFBundleIdentifier" "$INFO_PLIST")
APP_VERSION_IN_PLIST=$(/usr/libexec/PlistBuddy -c "Print CFBundleShortVersionString" "$INFO_PLIST")
BUILD_NUMBER_IN_PLIST=$(/usr/libexec/PlistBuddy -c "Print CFBundleVersion" "$INFO_PLIST")

if [ "$BUNDLE_ID" != "com.api-inquiry.APIInquiry" ]; then
    echo "Unexpected bundle id: $BUNDLE_ID" >&2
    exit 1
fi

if [ "$APP_VERSION_IN_PLIST" != "$APP_VERSION" ]; then
    echo "Unexpected app version: $APP_VERSION_IN_PLIST" >&2
    exit 1
fi

if [ "$BUILD_NUMBER_IN_PLIST" != "$BUILD_NUMBER" ]; then
    echo "Unexpected build number: $BUILD_NUMBER_IN_PLIST" >&2
    exit 1
fi
```

# API Inquiry v0.1.1-alpha.1

API Inquiry v0.1.1-alpha.1 is a polish release for the DeepSeek balance menu bar app.

## Highlights

- Clearer first-run setup for configuring a DeepSeek API key.
- Actionable refresh failure states with retry or key replacement actions.
- Safer API key management with delete confirmation and save feedback.
- Refresh button state now reflects in-progress refreshes.
- Versioned DMG artifact and SHA-256 checksum.

## Download

- `API-Inquiry-v0.1.1-alpha.1.dmg`
- `API-Inquiry-v0.1.1-alpha.1.dmg.sha256`

## Verify Download

```bash
shasum -a 256 -c API-Inquiry-v0.1.1-alpha.1.dmg.sha256
```

## Install

1. Download the DMG.
2. Open the DMG.
3. Drag `API Inquiry.app` into `Applications`.
4. Open API Inquiry from Applications.

This app is ad-hoc signed but not Apple notarized. On first launch, macOS may require right-clicking the app and choosing `Open`, or allowing the app in `System Settings > Privacy & Security`.

## Known Limits

- No Apple notarization.
- No detailed usage charts.
- No local DeepSeek usage console yet.
- DeepSeek is the only visible provider in this alpha.
```

# API Inquiry v0.1.1-alpha.1

API Inquiry v0.1.1-alpha.1 是 DeepSeek 余额菜单栏应用的体验打磨版本。

- 优化首次启动时的 DeepSeek API Key 配置引导。

- `API-Inquiry-v0.1.1-alpha.1.dmg.sha256`

```bash
shasum -a 256 -c API-Inquiry-v0.1.1-alpha.1.dmg.sha256

1. 下载 DMG。

3. 将 `API Inquiry.app` 拖入 `Applications`。
