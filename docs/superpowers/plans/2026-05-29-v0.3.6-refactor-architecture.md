# v0.3.6 Architecture Refactor Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Refactor API Inquiry into a cleaner built-in multi-provider architecture while preserving current behavior and UI.

**Architecture:** Centralize built-in provider business metadata in Core, keep provider visual metadata in the App target, make production ViewModels coordinator-only, split the Console SwiftUI file into focused components, and keep each behavior change protected by the existing executable test runner.

**Tech Stack:** Swift 5.9 package, SwiftUI/AppKit on macOS, Combine, local executable test harness `APIInquiryCoreTestsRunner`.

---

## File Structure

- Create: `Sources/APIInquiryCore/Providers/ProviderDescriptor.swift` when descriptor growth makes `ProviderCatalog.swift` too broad.
- Create: `Sources/APIInquiryCore/Providers/ProviderRegistration.swift` for `descriptor + makeProvider` built-in registrations.
- Create: `Sources/APIInquiryCore/Providers/BuiltInProviderRegistry.swift` so catalog order, default provider, and app provider construction share one source.
- Modify `Sources/APIInquiryCore/Providers/ProviderCatalog.swift` for richer descriptors and metadata enums.
- Modify `Sources/APIInquiryCore/Providers/BalanceProvider.swift` so provider implementations no longer own presentation metadata.
- Modify `Sources/APIInquiryCore/Providers/DeepSeekBalanceProvider.swift`, `ZhipuCodingPlanProvider.swift`, and `CodexQuotaProvider.swift` to keep only fetch/parse/error behavior.
- Modify `Sources/APIInquiryCore/Refresh/MultiProviderBalanceCoordinator.swift` to own descriptor-backed runtimes and concurrent refresh.
- Modify `Sources/APIInquiryCore/ViewModels/MenuBarBalanceViewModel.swift` and `UsageConsoleViewModel.swift` to remove production single-provider routing.
- Split `Sources/APIInquiryCore/Formatting/ProviderDisplayFormatter.swift` into focused formatter/tone files.
- Create App-layer visual/style files under `Sources/APIInquiryApp/`.
- Split `Sources/APIInquiryApp/UsageConsoleView.swift` into focused Console component files.
- Update tests in `Sources/APIInquiryCoreTestsRunner/`.
