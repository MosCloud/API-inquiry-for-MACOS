# API Inquiry v0.3.0 Multi-Provider Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the v0.3.0 multi-provider MVP with DeepSeek balance and Zhipu GLM Coding Plan usage, a Primary Provider-only menu bar, all-provider details, and Console-based provider management.

**Architecture:** Keep `BalanceRefreshController` as a single-provider state machine. Add a provider catalog, plan usage snapshot, provider runtime/coordinator, and local preference storage. View models derive menu bar, details panel, and Console display models from the coordinator.

**Tech Stack:** Swift 5.9, SwiftUI, AppKit, macOS Keychain, UserDefaults, Swift Package Manager, local `APIInquiryCoreTestsRunner`.
