# v0.3.7 Versioned UI Effects Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add restrained higher-version SwiftUI effects while preserving macOS 13+ as the supported fallback path.

**Architecture:** Introduce a small App-target versioned effects helper so availability checks stay centralized instead of scattered through feature views. Use higher-version APIs only where they clarify status, numeric changes, or result feedback. Keep macOS 13 behavior visually stable and avoid layout redesign.

**Tech Stack:** Swift 5.9, SwiftUI/AppKit, macOS 13+, Swift Package Manager, `APIInquiryCoreTestsRunner`, App target build verification.

---

## Scope Boundary

This pass includes:

- Numeric text transitions for primary amount and quota amount text.
- High-version refresh symbol effects with macOS 13 fallback.
- High-version sensory feedback for settings/result feedback with macOS 13 fallback.
- Documentation/progress updates and agent QA review.

This pass excludes:

- Raising the deployment target above macOS 13.
- Liquid Glass / `glassEffect`.
- Console layout redesign, new cards, new navigation, or dashboard behavior.
- Provider expansion, dynamic providers, credential storage changes, or v0.4.0 work.
- Any animation that distracts from quota, status, provider, or credential content.
