# v0.3.7 Liquid Glass UI 探索提示词

这份提示词用于在新的会话窗口中探索 API Inquiry 的 Liquid Glass / macOS material 视觉方向。它不是正式实现计划，也不是重设计授权。探索目标是判断“玻璃/材质”是否能服务内容，而不是让 UI 变得更花。

## 新窗口提示词

```text
我们现在在 /Users/zbw/Desktop/API-inquiry/.worktrees/v0.3.7 上探索 API Inquiry v0.3.7 的 Liquid Glass / macOS material UI 方向。

请先读取：
- AGENT.md
- task_plan.md
- findings.md
- progress.md
- docs/superpowers/plans/2026-05-30-v0.3.7-ui-polish_zh.md
- docs/superpowers/specs/2026-05-30-v0.3.7-liquid-glass-ui-exploration-prompt_zh.md

核心原则：
- 这次是视觉探索，不是正式重设计。
- API Inquiry 的 UI 必须保持极简、克制、高效、轻科技。
- UI 必须为内容服务。任何玻璃、半透明、模糊、反光、边缘高光、动画或材质效果，只要降低状态、额度、供应商、凭据内容的扫读效率，就要删除或拒绝。
- 菜单栏面板是快速查看状态的表面，不是 dashboard。
- Console 是聚焦的管理表面，不是运维后台。
- Liquid Glass / material 只能退到背景层，不能抢前景内容。

探索目标：
1. 判断 macOS material / Liquid Glass 风格是否适合 API Inquiry。
2. 找到最小可行的玻璃化位置，而不是整体玻璃化。
3. 保持现有信息结构、核心布局、供应商工作流和 Provider 架构不变。
4. 输出可回退、可审查的小范围改动建议或实验实现。

优先探索范围：
- Console window 外层背景或 root container 的轻量 material。
- Console 顶部导航条的轻量 material 或更原生的层级处理。
- 弹出层、确认层、菜单类表面是否适合使用系统 material。
- Provider 卡片、额度数字、状态 badge、API Key 操作区原则上保持稳定实底或足够对比度，不优先玻璃化。
- 菜单栏面板先谨慎评估，不要第一步就大改。

明确禁止：
- 不要做大面积玻璃卡片。
- 不要让文字或额度数字漂浮在复杂透明背景上。
- 不要加入强折射、强发光、明显边缘高光、装饰阴影或概念稿式效果。
- 不要为了玻璃效果降低信息密度、对比度或可读性。
- 不要把 Console 做成视觉看板。
- 不要改变 Provider 架构、凭据存储、刷新逻辑或 v0.4.0 多供应商能力。
- 不要为了新 UI API 直接提升最低支持系统版本；v0.3.7 默认仍保持 macOS 13+。可以使用 availability 分支给 macOS 14/15+ 做增强，但必须有 macOS 13 fallback。

建议工作方式：
1. 先阅读当前 UI 文件，理解 MenuBar、Console Home、Console API、Settings 的层级。
2. 先给出 2-3 个 Liquid Glass 探索方案，并说明取舍。
3. 推荐从最克制方案开始：只尝试 Console 外层和导航层 material，Provider 卡片暂时不动。
4. 如果需要实现，做成小 commit 或小 patch，确保随时可以回退。
5. 实现后运行 swift build；如果涉及 Core display state 或测试覆盖，再运行 swift run APIInquiryCoreTestsRunner。
6. 拉起应用给我审查，重点看深色/浅色、复杂桌面背景、Increase Contrast、Reduce Transparency 下的可读性。

验收标准：
- 一眼仍然是 API Inquiry，不是另一个 app。
- 内容比效果更突出。
- 状态和额度更容易读，不能更难读。
- 玻璃效果是背景层级，不是主角。
- 所有变更都能解释为“提升层级、扫读或反馈”，而不是“更酷”。
- 如果审查中发现可读性变差，优先回退玻璃效果，而不是继续调参美化。
```

## 设计理解摘要

- Liquid Glass 对 API Inquiry 的正确用法是“轻量环境质感”，不是“视觉主题”。
- 最合适的位置是窗口背景、导航层、弹出层这类容器表面。
- 最不适合的位置是核心数据、Provider 卡片主体、状态 badge、API Key 输入与确认区域。
- 探索成功的标志不是玻璃感明显，而是 UI 更有层级且内容仍然安静、清楚、可扫读。
