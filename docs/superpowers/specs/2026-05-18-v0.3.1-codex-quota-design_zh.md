# API Inquiry v0.3.1 Codex 额度设计

## 概要

v0.3.1 在现有多供应商基础上补充 Codex provider。Codex 不展示现金余额，也不使用 OpenAI Platform Admin Usage/Costs API；它按 Codex/ChatGPT 会话型额度查询逻辑展示当前 ChatGPT plan 下的 Codex quota。

菜单栏沿用当前紧凑图片标签设计：左侧使用 ChatGPT 图标，右侧展示 `5h xx%`，其中 `xx%` 表示 5 小时窗口剩余额度。详情页展示 Codex 的 5h 和 Week 两个剩余额度窗口。Console 额外展示当前 plan，例如 Free、Plus、Pro 5x、Pro 20x、Business、Enterprise/Edu。

## 目标

- 从 `main` 分支创建 v0.3.1 开发分支并保持变更小范围。
- 新增内置 Codex provider，并可在 Console 中添加、自动发现本机 Codex 登录态、设为 Primary Provider。
- 使用 ChatGPT/Codex 会话型 quota 查询路径，而不是普通 OpenAI API key 或 Admin API key。
- 菜单栏显示 ChatGPT 图标和 `5h xx%` 剩余额度。
- 菜单栏详情页展示 `5h` 和 `Week` 两个剩余额度窗口，以及对应 reset 时间。
- Console 的 Codex 摘要展示当前 plan。
- 保留 DeepSeek、智谱 GLM Coding Plan 的现有行为和 Keychain account。
- 自动化测试覆盖 Codex catalog、凭证解析、HTTP 请求、quota 解析、菜单栏展示、详情页 quota rows、Console plan 展示。

## 非目标

- 不实现 OpenAI Platform API 成本、token 用量、组织 usage 或 billing 查询。
- 不抓取网页 DOM，不做浏览器自动化。
- 不支持多 Codex 账号。
- 不做历史趋势、预算告警或额度消耗预测。
- 不把 ChatGPT session token、access token、account id 写入 UserDefaults、日志、文档或测试 fixture。

## Codex 查询语义

Codex provider 查询 ChatGPT backend 的会话型额度接口：

```text
GET https://chatgpt.com/backend-api/wham/usage
```

请求使用 ChatGPT/Codex access token：

```text
Authorization: Bearer <access_token>
Accept: application/json
ChatGPT-Account-Id: <account_id>   # 可选
```

Codex provider 默认不要求用户在 Console 手工输入 token。应用会优先自动读取本机 Codex 登录态：

- `$CODEX_HOME/auth.json`
- `~/.codex/auth.json`

本机 Codex auth JSON 中常见字段：

```text
tokens.access_token
tokens.account_id
```

如果自动发现失败，Codex provider 才使用 Keychain 中 `codex-session-token` 的手工 fallback。fallback 可以保存以下任一形式：

- 原始 access token。
- `Bearer <access token>`。
- 从本地 Codex auth 导出的 JSON，其中包含 `tokens.access_token` 或 `accessToken`，可选包含 `tokens.account_id`、`account_id` 或 `accountID`。

自动发现的 auth 文件不会被复制到 UserDefaults，也不会写入日志或文档。发送请求前只在内存中提取 access token 和可选 account id。应用不会删除或修改用户的 Codex auth 文件；Console 的删除操作只影响手工 fallback。

## 数据模型

新增 quota usage 快照，而不是把 Codex 塞进单窗口 `PlanUsageSnapshot`。

```swift
public enum ProviderDetailKind: Equatable {
    case balance
    case planUsage
    case quotaUsage
}

public struct QuotaWindowSnapshot: Equatable {
    public let label: String
    public let remainingPercentage: Decimal
    public let resetAt: Date?
    public let isAvailable: Bool
}

public struct QuotaUsageSnapshot: Equatable {
    public let providerID: ProviderID
    public let planName: String
    public let windows: [QuotaWindowSnapshot]
    public let fetchedAt: Date
}
```

`ProviderSnapshot` 增加 `.quotaUsage(QuotaUsageSnapshot)`。`BalanceState` 增加 `lastQuotaUsageSnapshot` 便于 ViewModel 读取。

窗口解析规则：

- `rate_limit.primary_window` 映射为 `5h`。
- `rate_limit.secondary_window` 映射为 `Week`。
- `remainingPercentage = 100 - used_percent`，并限制在 `0...100`。
- 优先使用 `reset_at` epoch seconds；缺失时使用 `reset_after_seconds + now()`。
- 缺少两个 quota window 时返回 `missingBalanceInfo`。

Plan 解析规则：

- `free` -> `Free`
- `plus` -> `Plus`
- 包含 `5x` 的 Pro 变体 -> `Pro 5x`
- 包含 `20x` 的 Pro 变体 -> `Pro 20x`
- `pro` -> `Pro`
- `business` / `team` / usage based business -> `Business`
- `enterprise` / `edu` / `education` -> `Enterprise/Edu`
- 其他非空值保留并做基本 title 化；缺失时显示 `Unknown`

## UI 展示

### 菜单栏

Codex 作为 Primary Provider 时，菜单栏 label 沿用当前图片渲染通道：

- 左侧：ChatGPT 图标。
- 右侧：`5h xx%`。
- 不显示 `CDX`、`GPT` 或其他额外 provider prefix。

如果 5h window 缺失但有其他 window，菜单栏回退到第一个 window。没有可用 snapshot 时显示 `Setup` 或 `--`。

### 菜单栏详情页

Codex 作为 Primary Provider 时：

- 顶部使用 ChatGPT/Codex 标识。
- 主数值显示 5h 剩余额度，例如 `5h 72% remaining`。
- 下方显示两个紧凑 quota rows：
  - `5h`、剩余额度、reset 时间。
  - `Week`、剩余额度、reset 时间。
- 状态文案为 `Quota available` 或 `Quota exhausted`。

Codex 作为 secondary provider 时，详情行显示 `5h xx%`、状态和最后刷新时间。

### Console

Home 的 provider summary 增加 Codex plan 指标。Codex 卡片至少显示：

- API Key 状态。
- 状态。
- Detail：`5h xx%`。
- Plan：例如 `Plus`、`Pro 5x`、`Pro 20x`。
- Updated。

API 页面继续使用现有 per-provider 管理面板，但 Codex 优先显示自动发现状态，例如 `Detected from Codex login`。只有自动发现失败时才展示手工 fallback 输入。反馈文案不暴露 token、auth file 路径细节或 account id。

## 错误处理

- HTTP 401/403 映射为 `authenticationFailed`，提示重新保存 Codex session 凭证。
- HTTP 429 映射为 `rateLimited`。
- 其他非 2xx 映射为 `serverError(statusCode:)`。
- JSON 结构无法解析映射为 `decodingFailed`。
- 缺少 quota window 映射为 `missingBalanceInfo`。
- 返回 HTML 或 Cloudflare 页面时按不可解码响应处理，不记录响应体。

## 测试

- Provider catalog 包含 `.codex`，顺序为 DeepSeek、Zhipu、Codex，且账号/prefix 唯一。
- Codex credential source 优先从 `$CODEX_HOME/auth.json` 或 `~/.codex/auth.json` 读取登录态；缺失时才回退到 Keychain 中的 raw token、Bearer token 或 auth JSON。
- Codex provider 对 `/backend-api/wham/usage` 发送 GET、Bearer header、可选 `ChatGPT-Account-Id`。
- Codex provider 解析 plan、5h 剩余额度、Week 剩余额度和 reset 时间。
- 错误状态映射覆盖 401、403、429、非 2xx、 malformed JSON、缺少 quota window。
- MenuBar view model 对 Codex primary 显示 `5h xx%`，详情 rows 暴露 5h 和 Week。
- UsageConsole view model 对 Codex summary 展示 plan。
- SwiftUI 层能选择 ChatGPT 图标资源或文字兜底，不影响 DeepSeek/智谱。

## 安全与隐私

Codex 自动发现只读取本机 auth 文件，不修改、不删除、不复制到 UserDefaults。手工 fallback 凭证只进入 Keychain。测试使用假 token 和 mock HTTP client。文档、源码、测试 fixture 不包含真实 token。错误文案不能包含 Authorization header、access token、session token、account id、auth file 内容或原始响应体。
