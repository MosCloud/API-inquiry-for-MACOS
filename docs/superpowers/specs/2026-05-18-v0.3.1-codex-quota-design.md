# API Inquiry v0.3.1 Codex Quota Design

## Summary

v0.3.1 adds a Codex provider to the existing multi-provider app. Codex does not show cash balance and does not use the OpenAI Platform Admin Usage/Costs API. It follows the Codex/ChatGPT session quota flow and shows the quota available under the user's current ChatGPT plan.

The menu bar keeps the current compact image label design: a ChatGPT icon on the left and `5h xx%` on the right, where `xx%` means remaining quota in the 5-hour Codex window. The detail panel shows both 5h and Week remaining quota windows. The Console also shows the current plan, such as Free, Plus, Pro 5x, Pro 20x, Business, or Enterprise/Edu.

## Goals

- Create v0.3.1 work from `main` and keep the change narrowly scoped.
- Add a built-in Codex provider that can be added, auto-detect local Codex login state, and be selected as the Primary Provider from the Console.
- Use the ChatGPT/Codex session quota path, not a regular OpenAI API key or Admin API key.
- Show the ChatGPT icon and `5h xx%` remaining quota in the menu bar.
- Show `5h` and `Week` remaining quota windows, plus reset times, in the menu detail panel.
- Show the current plan in the Codex Console summary.
- Preserve existing DeepSeek and Zhipu GLM Coding Plan behavior and Keychain accounts.
- Cover Codex catalog, credential parsing, HTTP request construction, quota parsing, menu display, detail rows, and Console plan display with automated tests.

## Non-Goals

- Do not implement OpenAI Platform API costs, token usage, organization usage, or billing queries.
- Do not scrape DOM or automate a browser.
- Do not support multiple Codex accounts.
- Do not add historical trends, budget alerts, or quota burn-rate prediction.
- Do not write ChatGPT session tokens, access tokens, account ids, or authorization headers to UserDefaults, logs, docs, or test fixtures.

## Codex Query Semantics

The Codex provider queries the ChatGPT backend session quota endpoint:

```text
GET https://chatgpt.com/backend-api/wham/usage
```

Requests use a ChatGPT/Codex access token:

```text
Authorization: Bearer <access_token>
Accept: application/json
ChatGPT-Account-Id: <account_id>   # optional
```

The Codex provider does not require the user to type a token into the Console by default. The app first auto-detects local Codex login state from:

- `$CODEX_HOME/auth.json`
- `~/.codex/auth.json`

Common fields in the local Codex auth JSON are:

```text
tokens.access_token
tokens.account_id
```

If auto-detection fails, the Codex provider falls back to the `codex-session-token` Keychain account. The fallback can store any of these forms:

- A raw access token.
- `Bearer <access token>`.
- Exported local Codex auth JSON containing `tokens.access_token` or `accessToken`, optionally with `tokens.account_id`, `account_id`, or `accountID`.

Auto-detected auth files are not copied to UserDefaults and are never logged or documented. The app extracts the access token and optional account id in memory before sending the request. It never deletes or modifies the user's Codex auth file; Console delete actions only affect manual fallbacks.

## Data Model

Add a quota usage snapshot instead of forcing Codex into the single-window `PlanUsageSnapshot`.

```swift
public enum ProviderDetailKind: Equatable {
    case balance
    case planUsage
    case quotaUsage
}

public struct QuotaWindowSnapshot: Equatable {
    public let label: String
    public let remainingPercentage: Decimal
    public let resetAt: Date?
    public let isAvailable: Bool
}

public struct QuotaUsageSnapshot: Equatable {
    public let providerID: ProviderID
    public let planName: String
    public let windows: [QuotaWindowSnapshot]
    public let fetchedAt: Date
}
```

`ProviderSnapshot` adds `.quotaUsage(QuotaUsageSnapshot)`. `BalanceState` adds `lastQuotaUsageSnapshot` for ViewModel access.

Window parsing rules:

- `rate_limit.primary_window` maps to `5h`.
- `rate_limit.secondary_window` maps to `Week`.
- `remainingPercentage = 100 - used_percent`, clamped to `0...100`.
- Prefer `reset_at` epoch seconds; otherwise use `reset_after_seconds + now()`.
- Return `missingBalanceInfo` when both quota windows are absent.

Plan parsing rules:

- `free` -> `Free`
- `plus` -> `Plus`
- Pro variants containing `5x` -> `Pro 5x`
- Pro variants containing `20x` -> `Pro 20x`
- `pro` -> `Pro`
- `business`, `team`, or usage-based business -> `Business`
- `enterprise`, `edu`, or `education` -> `Enterprise/Edu`
- Preserve other non-empty values with basic title casing; show `Unknown` when missing.

## UI Display

### Menu Bar

When Codex is the Primary Provider, the menu bar label uses the existing image rendering path:

- Left: ChatGPT icon.
- Right: `5h xx%`.
- No `CDX`, `GPT`, or extra provider prefix.

If the 5h window is missing but another window is present, the menu bar falls back to the first window. Without a usable snapshot it shows `Setup` or `--`.

### Menu Detail Panel

When Codex is the Primary Provider:

- The header uses a ChatGPT/Codex mark.
- The primary value shows 5h remaining quota, such as `5h 72% remaining`.
- A compact quota section below shows:
  - `5h`, remaining percentage, and reset time.
  - `Week`, remaining percentage, and reset time.
- Status text is `Quota available` or `Quota exhausted`.

When Codex is a secondary provider, its detail row shows `5h xx%`, status, and last refresh time.

### Console

The Home provider summary adds a Codex plan metric. A Codex card shows at least:

- API key status.
- Status.
- Detail: `5h xx%`.
- Plan: for example `Plus`, `Pro 5x`, or `Pro 20x`.
- Updated.

The API page keeps the existing per-provider management panel, but Codex first shows the auto-detected state, such as `Detected from Codex login`. Manual fallback input is shown only when auto-detection fails. Feedback must not expose tokens, auth file details, or account ids.

## Error Handling

- HTTP 401/403 maps to `authenticationFailed` and asks the user to replace the Codex session credential.
- HTTP 429 maps to `rateLimited`.
- Other non-2xx responses map to `serverError(statusCode:)`.
- Undecodable JSON maps to `decodingFailed`.
- Missing quota windows map to `missingBalanceInfo`.
- HTML or Cloudflare responses are treated as undecodable/unavailable responses, and raw response bodies are not logged.

## Tests

- Provider catalog includes `.codex`, ordered DeepSeek, Zhipu, Codex, with unique accounts and prefixes.
- Codex credential source first reads login state from `$CODEX_HOME/auth.json` or `~/.codex/auth.json`; if absent, it falls back to raw token, Bearer token, or auth JSON stored in Keychain.
- Codex provider sends GET to `/backend-api/wham/usage` with Bearer auth and optional `ChatGPT-Account-Id`.
- Codex provider parses plan, 5h remaining quota, Week remaining quota, and reset times.
- Error mapping covers 401, 403, 429, non-2xx, malformed JSON, and missing quota windows.
- MenuBar view model displays `5h xx%` for Codex primary and exposes detail rows for 5h and Week.
- UsageConsole view model shows the Codex plan in provider summaries.
- The SwiftUI layer can choose ChatGPT icon assets or a text fallback without affecting DeepSeek/Zhipu.

## Security And Privacy

Codex auto-discovery only reads the local auth file; it does not modify, delete, or copy it into UserDefaults. Manual fallback credentials live only in Keychain. Tests use fake tokens and mocked HTTP clients. Docs, source, and test fixtures contain no real tokens. Error text must never contain Authorization headers, access tokens, session tokens, account ids, auth file contents, or raw response bodies.
