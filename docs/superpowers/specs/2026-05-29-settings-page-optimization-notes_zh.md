# API Inquiry Settings 页面优化参考说明

## 摘要

Settings 页后续会成为 API Inquiry 的应用级控制中心，但它不应该成为供应商管理的杂物间。它应保持克制，只承接影响整个 App 的偏好、发布信息、安全与隐私说明、诊断入口和未来高级能力。供应商的添加、展示策略、凭据管理和单个供应商状态仍然由 Home、API 和菜单栏详情页分别承担。

## 当前状态

当前 Settings 页只包含语言切换、版本号和项目主页入口。这个范围很轻，但信息架构仍比较临时：已有内容还没有明确分组，也没有为未来的通知、更新、安全、诊断等应用级能力预留稳定位置。

## 页面职责

Settings 页应负责应用级设置和信息展示：

- 全局偏好，例如语言、开机自启、刷新频率、菜单栏显示格式。
- 通知与提醒，例如余额偏低、额度告急、提醒阈值。
- 安全与隐私说明，例如密钥存储策略、本地配置说明、不会上传本地凭据的声明。
- 更新与发布信息，例如当前版本、检查更新、Release Notes、项目主页。
- 诊断与维护，例如导出脱敏诊断信息、打开日志目录、重置应用偏好。
- 高级与实验功能，例如未来自定义 Provider 或调试开关。

Settings 页不应负责供应商级管理：

- 添加、移除供应商仍属于 Home 或 API。
- Primary Provider / 菜单栏显示对象仍属于 Home。
- 更换 API 密钥、打开 OpenAI/Codex 本地配置仍属于 API。
- 单个供应商余额、额度、刷新状态仍属于 Home 或菜单栏详情页。

## 可能承载的设置

### 通用

- 语言：`Auto / 中文 / English`。
- 开机自启：当前在菜单栏详情页底部，未来可以在 Settings 中提供正式位置。
- 刷新频率：未来可支持 `5 / 10 / 15` 分钟等选项。
- 菜单栏显示格式：例如仅图标、图标加金额、图标加状态。

### 通知与提醒

- 余额偏低提醒开关。
- 额度告急提醒开关。
- 余额或额度阈值设置。

这类功能适合未来版本，但需要谨慎设计，避免 API Inquiry 变成复杂告警系统。

### 安全与隐私

- 简短说明 API 密钥存储在 macOS Keychain。
- 说明本地配置和认证文件不会被上传。
- 提供打开 Keychain Access 或相关本地配置位置的入口。
- 提供发布前隐私打包审计的说明或链接。

这部分应以“状态/说明 + 管理入口”为主，不应使用大段重复说教。

### 更新与发布

- 当前版本号。
- 检查更新入口，供 v0.5.0 自动更新能力使用。
- Release Notes。
- GitHub 项目主页。
- 已知限制，例如 Apple notarization 尚未启用。

### 诊断

- 导出脱敏诊断信息。
- 打开日志目录。
- 重置应用偏好。

诊断能力应放在页面底部或高级区域，避免普通用户误触。任何导出内容都必须排除 API Key、token、Keychain 内容和用户本地认证文件。

### 高级

- Provider 调试开关。
- 自定义 Provider 或 endpoint 的入口。
- 实验性功能开关。

高级能力应晚于基础 Settings 信息架构落地，避免过早增加复杂度。

## 推荐的首轮优化范围

下一次 Settings 页优化建议先聚焦信息架构和视觉结构，而不是立刻新增复杂功能：

- 保留现有语言切换。
- 保留版本号和项目主页。
- 增加“安全与隐私”说明模块。
- 增加“关于”或“发布信息”模块。
- 为未来通知、更新、诊断能力预留结构，但不急于实现完整功能。

首轮目标是搭好干净框架，让未来功能自然有位置，而不是一次性把所有可能设置都塞进去。

## 视觉方向

Settings 页应接近 macOS 设置页的分组列表，而不是大卡片堆叠或营销式说明页：

- 使用紧凑分组，例如“通用”“安全与隐私”“关于”。
- 每组内部使用行式布局：左侧标题与可选辅助说明，右侧控件。
- 控件优先使用系统化样式：segmented picker、toggle、button、menu。
- 组间距略大，组内行高紧凑。
- 避免大段说明文字，避免把基础安全原则重复成宣传文案。
- 保持整体安静、可扫描、低装饰。

## 设计原则

- Menu bar 详情页只负责状态查看和轻操作。
- Console Home 负责供应商总览和菜单栏显示策略。
- Console API 负责凭据、本地配置和供应商接入管理。
- Console Settings 负责应用级偏好、发布信息、安全隐私和诊断。
- 当灵活性与产品逻辑统一冲突时，优先保持清晰边界和克制体验。
