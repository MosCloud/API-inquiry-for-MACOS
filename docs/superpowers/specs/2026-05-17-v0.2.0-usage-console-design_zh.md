# API Inquiry v0.2.0 供应商控制台设计

## 概要

v0.2.0 继续保持 API Inquiry 小巧克制：本地控制台只聚焦 API 供应商状态和 API Key 管理。历史用量导入、本地用量存储和用量明细表格从发布范围中移除。

菜单栏仍然负责快速查看余额；控制台负责管理已经配置的 API 供应商。

## 目标

- 新增本地控制台窗口，只保留 `Home` 和 `API` 两个页面。
- `Home` 展示已配置供应商，包括 API Key 状态、生效状态和余额状态。
- `Home` 提供 `Add Provider` 按钮，作为后续添加供应商的入口。
- 将 API Key 配置、更换和删除放到 `API` 页面。
- 保持当前 DeepSeek 供应商可用，不在 v0.2.0 引入多供应商持久化。
- 从应用 UI 和应用代码路径中移除历史用量导入和展示。

## 非目标

- 不做历史用量导入。
- 不做用量图表、用量历史合并或本地用量明细存储。
- v0.2.0 不实现新增供应商流程。
- 不做云同步、账号体系或跨设备共享。
- 不引入 Developer ID 签名或 Apple notarization 公证。

## 控制台结构

### Home

`Home` 列出供应商状态。v0.2.0 只有 DeepSeek。

每个供应商行展示：

- 供应商名称。
- API Key 状态：`Configured` 或 `Not configured`。
- 生效状态：`Active`、`Checking`、`Invalid`、`Unavailable`、`Insufficient balance` 或 `Not configured`。
- 如果已有余额快照，则显示余额。

`Add Provider` 按钮目前打开 `API` 页面。它只是一个轻量入口，不引入供应商注册表或配置向导。

### API

`API` 列出供应商 API 管理面板。v0.2.0 只有 DeepSeek 面板。

面板支持：

- 保存 API Key。
- 替换 API Key。
- 删除 API Key，并进行内联确认。
- 安全反馈文案，不包含 key 或 Authorization header。

## 架构

`UsageConsoleViewModel` 在当前分支保留原名称，但职责收敛为：

- 暴露余额状态。
- 派生供应商摘要。
- 执行 API Key 保存、替换和删除。

用量导入模型、解析器、导入器、存储和测试从 active target 中删除。

## 测试

核心测试覆盖：

- 未配置供应商的状态摘要。
- 已配置且生效供应商的状态摘要。
- API Key 无效供应商的状态摘要。
- API Key 保存、保存失败和删除行为。
- 现有菜单栏、余额刷新、Keychain 和 provider 行为。

手动检查：

- 从菜单栏打开控制台后，菜单栏详情页自动消失。
- 控制台只显示 `Home` 和 `API`。
- `Home` 包含 `Add Provider`。
- DeepSeek API Key 保存、替换和删除仍可用。
