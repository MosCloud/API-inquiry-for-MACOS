# API Inquiry v0.3.9 Codex 手动重置信息设计

## 摘要

v0.3.9 优化 Console Home 中 OpenAI/Codex 供应商卡片的信息展示，新增 Codex rate-limit reset credits 的“手动重置”摘要。该信息只在 Console Home 展示，是 OpenAI 卡片的附属信息，不进入 Menubar 展示，也不进入 Menubar 或主 quota 的刷新链路。

本版本保持现有 5h/7d quota 查询和 Menubar 体验不变，只在 Console Home 的 OpenAI 卡片中用更有价值的手动重置信息替换低频变化的计划名。

## 目标

- 在 Console Home 的 OpenAI 卡片中新增“手动重置”指标。
- 用“手动重置”替换现有 OpenAI 卡片的“计划”指标。
- 保留 OpenAI 卡片现有的 `API 密钥`、`状态`、`详情`、`更新于` 指标。
- `详情` 继续显示 5h 剩余额度，例如 `5 时 98% 剩余`。
- “手动重置”展示可用重置卡摘要，例如 `2 张 · 7/18 到期`。
- 在“手动重置”标题行右侧提供一个小的 icon-only 刷新按钮。
- reset credits 查询复用本机 Codex 登录态和 ChatGPT backend endpoint。
- reset credits 查询是 Console Home-only、低频缓存、局部刷新、非阻塞的附属查询。
- 自动化测试覆盖数据解析、缓存/刷新边界、Console Home display model 和 UI 结构。

## 非目标

- 不修改 Menubar 展示。
- 不修改 Menubar 刷新语义。
- 不把手动重置信息显示在 Menubar 或 Menubar 详情页。
- 不把 reset credits 查询加入 `BalanceRefreshController` 主刷新链路。
- 不让 reset credits 查询阻塞 `/backend-api/wham/usage` 的 5h/7d 主 quota 刷新。
- 不新增 Console Home 全局刷新按钮。
- 不重做 OpenAI 卡片整体布局。
- 不影响 DeepSeek、智谱等 provider。
- 不实现“执行手动重置”的动作。
- 不做历史记录、告警、趋势、预测或到期通知。
- 不保存或展示 token、refresh token、cookie、完整 account id 或原始响应体。

## 信息展示

OpenAI 卡片保持现有横向 metric grid 的信息架构：

```text
API 密钥 | 状态 | 详情 | 手动重置 | 更新于
```

其中：

- `API 密钥`：保持现有配置/加载状态。
- `状态`：保持现有 quota 可用/不可用状态。
- `详情`：保持现有 5h 剩余额度文案。
- `手动重置`：新增 reset credits 摘要。
- `更新于`：保持现有主 quota 刷新时间。

`计划` 指标被替换，不再在 Home 卡片中显示。计划名变化低频，且当前 Home 空间有限；手动重置的可用张数和过期时间对用户更有操作价值。

## 手动重置摘要规则

reset credits payload 中的每张卡只保留必要字段：

- 发放时间。
- 过期时间。
- 是否已使用。

Console Home 只展示聚合摘要，不展示每张卡的完整明细。默认摘要规则：

- 可用卡数量为未使用且未过期的卡数量。
- 最近过期日取可用卡中最早的 `expires_at`。
- 有可用卡时显示：`N 张 · M/d 到期`。
- 没有可用卡时显示：`0 张`。
- 未查询或无缓存时显示：`--`。
- 查询中时显示：`查询中`。
- 无缓存且查询失败时显示：`查询失败`。
- 有缓存但刷新失败时保留旧摘要，不覆盖为失败。

英文界面使用对应英文文案，例如 `2 cards · exp. 7/18`、`Checking`、`Failed`。

日期显示遵循现有本地化和系统时区策略。内部解析以 UTC 时间为准，展示时转为本地时间；中文环境下通常表现为北京时间。

## 刷新按钮设计

“手动重置”指标支持一个可选 accessory icon button：

```text
手动重置        ↻
2 张 · 7/18 到期
```

设计要求：

- 按钮放在 metric 标题行右侧，不放在数值文本后方。
- 使用系统符号 `arrow.clockwise`。
- icon-only，不显示按钮文字。
- 尺寸控制在约 `20x20` 或 `22x22`，比 Menubar 顶部刷新按钮更低调。
- 使用 borderless/plain 风格。
- 默认使用 `.secondary` 颜色，hover 时轻微增强。
- tooltip 为 `刷新手动重置信息`。
- accessibility label 为 `刷新手动重置信息`。
- 查询中时只让该按钮 disabled 或局部旋转，不让整张 OpenAI 卡片进入 loading。
- 该按钮只刷新 reset credits，不刷新 5h/7d、其他供应商或 Menubar。

按钮不放在 OpenAI 卡片 header 右侧，避免和右侧健康状态 badge 抢视觉层级。按钮不放在值后方，避免长文本挤压和横向抖动。

## 数据获取

新增 Console Home 专用 reset credits 查询：

```text
GET https://chatgpt.com/backend-api/wham/rate-limit-reset-credits
Authorization: Bearer <access_token>
Accept: application/json
ChatGPT-Account-Id: <account_id>   # 如本机 auth 中存在，可复用现有逻辑
```

凭证来源复用现有 Codex 凭证模型：

- 优先读取本机 Codex auth 文件。
- 常见位置为 `$CODEX_HOME/auth.json` 和 `~/.codex/auth.json`。
- 使用 `tokens.access_token`。
- 可选使用 `tokens.account_id`。
- fallback 行为与现有 Codex quota 查询保持一致。

实现上不应复制一套新的 token 解析规则。应抽出或复用现有 `CodexQuotaProvider` 的 credential parsing 能力，保证 `/wham/usage` 和 `/wham/rate-limit-reset-credits` 的认证行为一致。

## 刷新逻辑

5h/7d quota 是主刷新链路；手动重置是 Console Home 的附属查询。

主刷新链路：

- Menubar 自动刷新继续只查询 `/backend-api/wham/usage`。
- Menubar 手动刷新继续只查询 `/backend-api/wham/usage`。
- `BalanceRefreshController` 继续只负责 provider 主 snapshot。
- reset credits 不进入 `ProviderSnapshot` 主刷新结果，不影响主状态 loading 和错误状态。

手动重置刷新链路：

- Console Home 出现时，如果 OpenAI 已配置且没有缓存，则后台查询 reset credits。
- Console Home 出现时，如果缓存已过期，则后台查询 reset credits。
- 缓存未过期时直接展示缓存，不发请求。
- 建议缓存有效期为 1 小时。
- 用户点击“手动重置”指标内的局部刷新按钮时，强制刷新 reset credits。
- 同一时间只允许一个 reset credits 查询在进行中，重复点击不启动并发请求。
- reset credits 查询有自己的 loading/error 状态，不改变 OpenAI 卡片的主 quota 状态。

## 错误处理

- HTTP 200：解析 reset credits 并更新缓存。
- HTTP 401/403：视为本机 Codex 凭证失效或 Authorization header 问题，只影响手动重置指标。
- HTTP 429：视为 reset credits 查询限流，只影响手动重置指标。
- 其他非 2xx：视为 reset credits 查询失败，只影响手动重置指标。
- JSON 无法解析：视为 reset credits 查询失败，只影响手动重置指标。
- 如果存在旧缓存，失败时保留旧缓存摘要。
- 如果没有旧缓存，失败时显示 `查询失败`。
- 错误文案不得包含 token、refresh token、cookie、完整 account id、Authorization header 或原始响应体。

## 架构边界

推荐新增或调整以下边界：

- `CodexCredential` 解析能力从 `CodexQuotaProvider` 私有实现中抽出为 Core 内部可复用 helper。
- 新增 `CodexManualResetCreditsProvider` 或同等轻量 client，专门负责 reset credits endpoint。
- 新增 reset credits display model，挂在 `UsageConsoleViewModel` 的 Console Home summary 上。
- `UsageConsoleHomeSection` 和 `ProviderMetricBox` 支持某个 metric 携带可选 accessory button。
- Menubar view model、Menubar view 和 `BalanceRefreshController` 不依赖 reset credits 数据。

实现应保持最小改动。不要为 reset credits 引入持久化存储、独立 provider 注册、全局刷新控制器或跨页面状态机。

## 测试

Core 测试：

- Codex credential helper 能解析 raw token、Bearer token 和 auth JSON。
- `/wham/usage` 仍使用相同 Authorization header 和可选 `ChatGPT-Account-Id`。
- reset credits client 对正确 endpoint 发送 GET、Bearer header 和 Accept header。
- reset credits client 解析 `credits[*].granted_at`、`expires_at`、`redeemed_at`。
- reset credits client 将 401/403 映射为鉴权失败，且不暴露敏感响应。
- display formatter 将可用卡聚合为 `N 张 · M/d 到期`。
- display formatter 在无可用卡、未查询、查询中、失败、有缓存失败时输出正确文案。
- `UsageConsoleViewModel` 的 reset credits 刷新不调用 `coordinator.refreshAddedProviders()`。
- reset credits 缓存未过期时不重复请求，局部刷新按钮强制请求。

App/UI 测试或可测试结构：

- OpenAI Home metrics 从 `计划` 改为 `手动重置`。
- `手动重置` metric 暴露 accessory refresh action。
- accessory button 使用 `arrow.clockwise`、tooltip/accessibility label 为 `刷新手动重置信息`。
- DeepSeek 和智谱 Home metrics 不出现该 accessory button。

回归测试：

- Menubar primary Codex 仍显示 5h quota。
- Menubar refresh 仍只触发主 provider 刷新。
- Console Home 主 OpenAI 详情仍显示 5h 剩余额度。
- 现有 507 条测试期望继续通过。

## 安全与隐私

- 不记录真实 access token、refresh token、cookie 或完整 account id。
- 不把 auth JSON 内容写入日志、文档或测试 fixture。
- 不展示原始 reset credits 响应。
- 测试只使用假 token 和 mock HTTP client。
- 用户可见错误只说明凭证失效、请求失败或查询失败，不包含敏感字段。
