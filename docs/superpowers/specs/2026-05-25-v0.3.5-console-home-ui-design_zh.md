# API Inquiry v0.3.5 Console Home UI 设计

## 摘要

v0.3.5 优化 Console Home 页，让它在视觉上更像菜单栏详情页所属的同一套产品语言，同时保留 Console 偏管理、偏具体信息的内容模型。本轮只聚焦 Home：供应商行变成带健康度染色的半透明单元，右上角状态胶囊使用同一个聚合健康度，Console 窗口本身也对齐详情页的 macOS 透明材质质感。

这是一个视觉和 display model 打磨版本。不改变 provider 拉取、凭据存储、API Key 管理、供应商添加/移除行为，也不重做 API 和 Settings 页面布局。

## 目标

- 保持 Console Home 是详细的供应商管理概览，而不是菜单栏详情页的放大版。
- 让 Home 供应商行对齐详情页设计语言：克制圆角、半透明层级、语义健康色、安静的次级文本。
- 增加一个 Home 专用的供应商健康度 tone，复用现有 `ProviderAmountTone` 阈值。
- 当供应商有多个额度窗口时，使用最紧急的额度或用量窗口作为整行 tone。
- Home 每个供应商行的背景色和右上角状态胶囊同步使用该聚合健康度。
- 让 Console 窗口使用更接近菜单栏详情页的 macOS 透明材质质感。
- 本版本保留 API 和 Settings 页面行为与布局。
- 用 Core 测试覆盖新的 display-model tone 规则。

## 非目标

- 不重设计 API 页面表单、API Key 确认流程或 Settings 页面。
- 不增加图表、历史记录、预算、预测或分析能力。
- 不修改 provider 刷新频率、网络解析、供应商排序、添加/移除规则或 primary provider 持久化。
- 不给 Home 中所有 metric 数字上色。健康信号由供应商行背景和状态胶囊承担。
- 不修改菜单栏标题、菜单栏详情页布局或详情页主数字颜色阈值。
- 不暴露 API Key，也不改变凭据存储。

## 设计方向

最终方向是 “A+C”：

- 保留当前 Console Home 的信息架构：供应商列表、供应商名称/Logo、API Key 状态、验证状态、供应商详情、重置/套餐信息、更新时间和菜单栏 primary 标记。
- 增加强视觉方案中的健康度染色供应商行背景。
- 右侧状态文案仍显示 `Available`、`Quota available`、`可用` 等状态本身，但胶囊颜色使用供应商行的聚合健康度。

这样 Home 仍然具体、可扫描，同时不会变成厚重的数据看板。

## Home 健康度规则

Home 在 `APIProviderSummary` 上增加新的 display-model 值，例如 `healthTone: ProviderAmountTone`。

### 余额类供应商

余额类供应商复用现有余额金额阈值：

- `totalBalance >= 50`：`.good`
- `totalBalance >= 10` 且 `< 50`：`.warning`
- `totalBalance < 10`：`.critical`

### Used Plan 类供应商

Used-plan 供应商复用现有用量百分比阈值：

- `usagePercentage <= 40`：`.good`
- `usagePercentage > 40` 且 `<= 80`：`.warning`
- `usagePercentage > 80`：`.critical`

### Remaining Quota 类供应商

Remaining-quota 供应商先给每个额度窗口按现有剩余额度阈值计算 tone：

- `remainingPercentage >= 60`：`.good`
- `remainingPercentage >= 20` 且 `< 60`：`.warning`
- `remainingPercentage < 20`：`.critical`

整张供应商行使用最紧急窗口的 tone。紧急程度顺序为：

```text
critical > warning > good > neutral
```

例如 Codex 的 `5h remaining = 18%`、`Weekly remaining = 63%` 时，Home 供应商行和右上角胶囊都使用 `.critical`。

### 非额度状态

只有在存在可用余额、used-plan 或 remaining-quota 数据时，健康度才接管颜色。没有可用额度语义的状态保留状态本身的含义：

- `Not configured`：中性行和中性胶囊。
- `Checking`：中性或刷新中处理，不套用额度健康色。
- `Invalid` 和 `Unavailable`：使用状态 warning/error 语义，而不是额度健康色。
- 失败状态如果保留了 last snapshot，可以继续显示 last snapshot 详情，但如果当前状态是鉴权或可用性失败，整行不应该看起来像健康状态。

## 视觉处理

### Console 窗口

Console 窗口使用透明/非 opaque 的 AppKit 外壳和 SwiftUI material 背景：

- 透明 title bar。
- 适当使用 full-size content view，让内容和标题栏属于同一透明窗口质感。
- window background 非 opaque。
- SwiftUI 根背景使用 macOS material，而不是纯实色。

实现应优先使用系统 material 和环境行为，而不是手写固定 alpha 值，这样 UI 可以跟随深浅色模式和“降低透明度”等辅助功能设置。

### Home 供应商行

每个 Home 供应商行：

- 保持 8px continuous 圆角。
- 使用半透明基础 fill。
- 添加轻微的健康色 leading wash 或 tint。
- metric 文本保持中性和可读。
- 当存在额度健康语义时，右上角状态胶囊与整行使用同一个健康度 tone。
- 非额度状态使用中性或状态 warning 处理。

### 导航和 Section Header

顶部 segmented navigation 可以保持结构不变，但应放在兼容新窗口 material 的半透明 fill 上。Home section title 和 Add Provider menu 保持原位。

## 架构

额度健康度判断属于 Core display formatting，不应该在 SwiftUI 中检查字符串。

推荐改动：

- 给 `APIProviderSummary` 增加 `healthTone: ProviderAmountTone`。
- 在 `ProviderDisplayFormatter` 增加 helper，从 `BalanceState` 计算 provider summary health tone。
- 保留现有 `ProviderStatusTone` 表达 neutral、refreshing、success、warning 等运行状态。
- 让 `UsageConsoleView` 将 `ProviderAmountTone` 映射成 Home 行背景和状态胶囊颜色。
- Console 窗口透明设置放在 `UsageConsoleWindowController`。

SwiftUI 不应该从 `Available`、`额度可用` 或 `18%` 等本地化字符串反推健康度。

## 数据流

1. Provider refresh 逻辑继续产生 `BalanceState`。
2. `UsageConsoleViewModel.providerSummaries` 构造每个 `APIProviderSummary`。
3. `ProviderDisplayFormatter.summaryHealthTone(for:)` 从当前 state 计算 Home 行健康度。
4. `UsageConsoleView.providerStatusRow(_:)` 使用 `summary.healthTone` 渲染 Home 行背景和右侧胶囊。
5. API 和 Settings section 保持当前控件和行为。

## 错误处理

- 缺少 snapshot 时返回 `.neutral`。
- loading 且没有 last snapshot 时返回 `.neutral`。
- loading 且有 last snapshot 时可以使用中性/刷新中 UI，而不是把 stale 健康状态当作当前健康状态呈现。
- 鉴权失败和不可用失败使用状态 warning 处理。
- 失败状态绝不在反馈或显示文本中暴露 API Key。

## 测试

Core 测试覆盖 display-model 规则：

- DeepSeek 余额边界：`50`、`49.99`、`10`、`9.99`。
- Used-plan 边界：`40`、`40.01`、`80`、`80.01`。
- Remaining-quota 聚合：`[60, 59, 20, 19]` 应聚合为 `.critical`。
- Remaining-quota 混合窗口：`5h = 18`、`Week = 63` 应聚合为 `.critical`。
- 非额度状态：未配置、无快照 loading、鉴权失败、不可用失败使用中性或状态 warning，而不是健康额度染色。
- Provider summaries 在单供应商和多供应商 Console 模式下暴露新的 tone。

手动验证：

- Console Home 使用与菜单栏详情页一致的半透明视觉语言。
- 供应商行背景和右上角状态胶囊同步变化。
- Home 仍是详细供应商概览，而不是 hero 化详情页。
- API 和 Settings 页面流程仍可使用，且本版本未重设计这些页面。
