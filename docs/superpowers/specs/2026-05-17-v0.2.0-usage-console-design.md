# API Inquiry v0.2.0 Provider Console Design

## Summary

v0.2.0 keeps API Inquiry small by focusing the local console on API provider status and API key management. Historical usage import, local usage storage, and usage detail tables are removed from the release scope.

The menu bar remains the fast balance surface. The console becomes the management surface for configured API providers.

## Goals

- Add a local console window with only two pages: `Home` and `API`.
- Show configured providers on `Home`, including API key status, validation status, and balance state.
- Include an `Add Provider` button on `Home` as the future entry point for additional providers.
- Move API key setup, replacement, and deletion into the `API` page.
- Keep the current DeepSeek provider working without adding multi-provider persistence yet.
- Remove historical usage import and display from the app UI and app code path.

## Non-Goals

- No historical usage import.
- No usage charts, usage history merge, or local usage detail storage.
- No new provider onboarding flow in v0.2.0.
- No cloud sync, accounts, or cross-device sharing.
- No Developer ID signing or Apple notarization.

## Console Structure

### Home

`Home` lists configured provider surfaces. In v0.2.0 the list contains DeepSeek only.

Each provider row shows:

- Provider name.
- API key status: `Configured` or `Not configured`.
- Validation status: `Active`, `Checking`, `Invalid`, `Unavailable`, `Insufficient balance`, or `Not configured`.
- Balance text when a balance snapshot exists.

The `Add Provider` button opens the `API` page for now. It is intentionally lightweight and does not introduce a provider registry or setup wizard yet.

### API

`API` lists provider API management panels. In v0.2.0 there is one DeepSeek panel.

The panel supports:

- Saving an API key.
- Replacing an API key.
- Deleting an API key with inline confirmation.
- Safe feedback that never includes the key or authorization header.

## Architecture

`UsageConsoleViewModel` is retained as the console view model name for this branch, but its responsibility is narrowed to:

- Balance state exposure.
- Provider summary derivation.
- API key save, replace, and delete actions.

The usage import models, parser, importer, storage, and tests are removed from the active target.

## Testing

Core tests cover:

- Provider summary for an unconfigured provider.
- Provider summary for a configured and active provider.
- Provider summary for an invalid API key.
- API key save, failed save, and deletion behavior.
- Existing menu bar, balance refresh, keychain, and provider behavior.

Manual checks:

- Console opens from the menu bar and closes the menu bar details window.
- Console shows `Home` and `API` only.
- `Home` includes `Add Provider`.
- Saving, replacing, and deleting the DeepSeek API key still works.
