# API Inquiry v0.3.0 多供应商设计

## 概要

v0.3.0 将 API Inquiry 从 DeepSeek 单供应商余额工具升级为多供应商状态工具。第一批真实供应商包含 DeepSeek 和智谱 GLM Coding Plan。

菜单栏仍然保持克制：只展示一个 Primary Provider 的高频状态。点击菜单栏后，详情页展示所有已添加供应商的状态。供应商的添加、API Key 管理和 Primary Provider 设置继续放在 Console 中。

## 目标

- 支持内置供应商目录，第一批包含 DeepSeek 和智谱 GLM Coding Plan。
- 保持 DeepSeek 现有 Keychain account、刷新行为、余额展示和升级兼容性。
- 为智谱 GLM Coding Plan 增加真实 plan 用量查询能力。
- 菜单栏只显示 Primary Provider 的详情。
- 菜单栏详情页展示所有供应商的详情。
- Console 继续作为供应商管理入口，管理 API Key、添加供应商和设置 Primary Provider。
- 每个供应商独立保存 API Key、刷新状态、最后一次快照和错误状态。
- 自动化测试覆盖多供应商状态隔离、Keychain account 隔离、Console 管理、菜单栏展示和智谱解析。

## 非目标

- 不做任意自定义供应商、动态 endpoint 配置、脚本插件或 provider marketplace。
- 不做历史用量导入、用量图表、成本分析或预算告警。
- 不做跨币种余额聚合、汇率换算或多供应商总额。
- 不做同一供应商多账号或多 API Key 管理。
- 不做云同步、账号体系、团队共享或跨设备状态同步。
- 不抓取网页控制台，不做浏览器自动化。
- 不引入 Developer ID 签名或 Apple notarization。
- 不把菜单栏详情页变成管理后台；管理能力继续留在 Console。

## 供应商范围

### DeepSeek

DeepSeek 继续作为默认供应商和默认 Primary Provider。

DeepSeek 展示的是余额：

- 菜单栏：紧凑余额，例如 `¥68.6`。
- 详情页顶部：沿用当前 DeepSeek 余额的大号展示，例如 `¥68.65 CNY`。
- 次级行：显示供应商名称、余额、可用状态和最后刷新时间。

DeepSeek 必须继续使用现有 Keychain account：

```text
deepseek-api-key
```

### 智谱 GLM Coding Plan

智谱 GLM Coding Plan 是 v0.3.0 的第一个新增真实供应商。

智谱展示的是 plan 用量，而不是现金余额：

- 菜单栏：显示窗口和用量百分比，例如 `5h 17%`。
- 详情页顶部：当智谱是 Primary Provider 时，显示 `5h` 和 `17% used`，只放大数字 `17`。
- 次级行：显示供应商名称、窗口、用量百分比、可用状态、下一次重置时间和最后刷新时间。

智谱官方文档确认：

- GLM Coding Plan 使用专属 Coding endpoint：`https://open.bigmodel.cn/api/coding/paas/v4`。
- API 使用 HTTP Bearer 鉴权：`Authorization: Bearer YOUR_API_KEY`。
- 错误码包含 GLM Coding Plan 相关状态，例如套餐到期 `1309`、达到使用上限并等待重置的 `1308` 或 `1310`。

额度查询 endpoint 在实现前必须做本地验证。公开资料中常见以下候选 endpoint：

```text
https://open.bigmodel.cn/api/monitor/usage/quota/limit
https://bigmodel.cn/api/monitor/usage/quota/limit
```

验证只允许使用本机输入或 Keychain 中的真实 key，不能把真实 API Key 写进聊天、文档、源码、日志或测试 fixture。

## Primary Provider 规则

Primary Provider 是菜单栏展示的第一供应商。

- 默认 Primary Provider 是 DeepSeek。
- Console 中提供 `Show in Menu Bar` 或等价操作，用于设置 Primary Provider。
- 菜单栏 label 只显示 Primary Provider 的自适应详情。
- 如果 Primary Provider 未配置 key，菜单栏显示该供应商的 setup 状态。
- 如果 Primary Provider 刷新失败但存在上一条快照，菜单栏继续显示上一条可用快照，并在详情页展示错误状态。
- 详情页顶部始终展示 Primary Provider。
- 详情页下方用紧凑行展示其他已添加供应商。

## 菜单栏详情页

### 顶部 Primary Provider 区域

顶部区域保持当前 DeepSeek 详情页的视觉层级：供应商标识、主要数值、状态、最后刷新时间和快捷操作。

余额型供应商：

- 主要数值是余额。
- 货币符号和货币代码保持较弱层级。
- 示例：`¥68.65 CNY`。

Plan 型供应商：

- 主要数值是用量百分比中的数字。
- 窗口和单位作为辅助信息。
- 示例：`5h`、`17% used`，其中 `17` 是最大字号。

### 其他供应商行

每个非 Primary Provider 占一行。

每行展示：

- 供应商名称。
- 自适应详情：余额或 plan 用量。
- 可用状态：例如 `Available`、`Balance insufficient`、`Plan available`、`Limit reached`、`Invalid`、`Unavailable`、`Not configured`。
- 最后刷新时间。
- 可选的下一次重置时间，适用于 plan 型供应商。

详情页可以提供刷新按钮和 Console 按钮，但不提供 API Key 编辑、删除 provider 或添加 provider。

## Console 管理

Console 继续包含 `Home` 和 `API` 页面。

### Home

Home 展示已添加供应商列表。

每个供应商摘要显示：

- 名称。
- API Key 状态。
- 验证状态。
- 余额或 plan 用量。
- 最后刷新时间。
- 是否为 Primary Provider。

`Add Provider` 打开内置供应商选择器，只展示尚未添加的内置供应商。

### API

API 页面展示 per-provider 管理面板。

每个供应商面板支持：

- 保存 API Key。
- 替换 API Key。
- 删除 API Key，并进行内联确认。
- 设置为 Primary Provider。
- 从已添加供应商列表中移除供应商。

删除供应商时，如果该供应商有保存的 API Key，必须明确提示会同时删除该 key。确认后删除该供应商的 Keychain item，并从本地已添加列表中移除。

DeepSeek 不能从内置目录中消失。即使删除 DeepSeek API Key，DeepSeek 仍保留为可添加或默认供应商，以保证升级兼容和回退路径。

## 架构

### Provider Catalog

新增 provider catalog，用于集中声明内置供应商、默认顺序和默认 Primary Provider。

每个 provider descriptor 包含：

- `ProviderID`
- display name
- menu prefix
- credential account
- homepage URL
- detail kind：balance 或 plan usage
- provider factory

ProviderID 在 v0.3.0 至少包含：

```swift
case deepseek
case zhipuCodingPlan
```

### 单供应商 Controller

`BalanceRefreshController` 保持单供应商状态机，不直接改造成多供应商 controller。

这样可以保留当前已测试的行为：

- 一个 provider。
- 一个 credential account。
- 一个 `BalanceState`。
- 一个自动刷新 task。
- 失败时保留该 provider 的上一条快照。
- 防止同一 provider 的重叠刷新。

### 多供应商 Coordinator

新增多供应商 coordinator 或 runtime 集合，用于组合多个单供应商 controller。

职责包括：

- 按 `ProviderID` 获取供应商 runtime。
- 刷新单个供应商。
- 刷新所有已配置供应商。
- 启动和停止所有供应商的自动刷新。
- 暴露各供应商的状态和最后快照。

多供应商 coordinator 不负责 API Key 明文保存，也不解析 UI 文案。

### Provider Detail Model

新增统一的 provider detail display model，用于表示余额型和 plan 型详情。

余额型 detail 包含：

- total balance
- currency
- availability
- fetched at

Plan 型 detail 包含：

- window label，例如 `5h`
- usage percentage，例如 `17`
- usage text，例如 `17% used`
- reset time
- availability
- fetched at

菜单栏、详情页和 Console summary 都应从这个 display model 派生展示，而不是通过状态字符串反推颜色或布局。

### 本地偏好

API Key 只存 Keychain。非敏感偏好可以存在本地偏好存储中，例如 UserDefaults。

可存储的信息：

- 已添加 provider IDs。
- Primary Provider ID。

不可存储的信息：

- API Key。
- Authorization header。
- API 响应中的敏感字段。

## 错误处理

所有供应商共享基础状态：

- `Not configured`
- `Checking`
- `Available`
- `Invalid`
- `Unavailable`

余额型供应商额外支持：

- `Balance insufficient`

Plan 型供应商额外支持：

- `Plan available`
- `Limit reached`
- `Plan expired`

智谱错误映射建议：

- HTTP 401 或业务认证错误：`Invalid`
- HTTP 429、业务错误 `1302`、`1303`、`1305`：`Unavailable` 或 `Limit reached`，按响应语义区分
- 业务错误 `1308`、`1310`：`Limit reached`，如果响应包含 reset time，则展示重置时间
- 业务错误 `1309`：`Plan expired`
- 5xx：`Unavailable`
- 解码失败：`Unavailable`，并保留上一条快照

保存 API Key 后如果刷新失败，沿用当前行为：key 仍保存，但展示 warning 反馈，状态显示为 `Invalid` 或 `Unavailable`。

所有错误反馈不得包含 API Key、Authorization header 或原始敏感响应。

## 测试

自动化测试使用现有 `APIInquiryCoreTestsRunner`。

新增或更新测试覆盖：

- provider catalog 默认顺序、唯一 ID、唯一 credential account、唯一 menu prefix。
- DeepSeek 仍使用 `deepseek-api-key`。
- 智谱 provider 请求 URL、HTTP method、Authorization header 和响应解析。
- 智谱 plan 用量快照解析，包括窗口、百分比、reset time 和可用状态。
- 智谱错误码映射，包括认证失败、限流、达到上限、套餐到期和服务错误。
- 多 provider credential 隔离：保存、替换、删除一个 key 不影响另一个 provider。
- 多 provider runtime 隔离：一个 provider 刷新失败不污染另一个 provider 的 state 或 last snapshot。
- Console summary 展示多个 provider。
- Console per-provider API Key 保存、替换、删除和反馈。
- Primary Provider 设置和恢复。
- 菜单栏 label 对余额型和 plan 型 provider 的格式化。
- 菜单栏详情页 hero model 和 secondary rows model。
- API Key 不出现在反馈、日志或测试失败信息中。

验证命令：

```bash
swift run APIInquiryCoreTestsRunner
swift build
Scripts/build-local-app.sh
```

## 手动检查

- 无任何 key 启动时，DeepSeek 是默认 Primary Provider，菜单栏显示 setup 状态。
- Console 可添加智谱 GLM Coding Plan。
- DeepSeek 保存 key 后，菜单栏和详情页显示余额。
- 智谱保存 key 后，Console 和详情页显示 plan 用量。
- 将智谱设为 Primary Provider 后，菜单栏显示 `5h 17%` 这类 plan 用量。
- 智谱作为 Primary Provider 时，详情页顶部放大 `17`，并展示 `5h` 和 `% used`。
- DeepSeek 作为 Primary Provider 时，详情页顶部保持当前余额展示风格。
- 非 Primary Provider 在详情页以一行展示所有状态信息。
- 删除智谱 key 不影响 DeepSeek key、余额和菜单栏状态。
- 智谱达到上限、套餐过期或 key 无效时，状态文案准确且不泄露 key。
