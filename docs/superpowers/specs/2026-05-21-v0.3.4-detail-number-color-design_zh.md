# API Inquiry v0.3.4 详情页数字颜色设计

## 摘要

v0.3.4 优化菜单栏展开详情页的主模块视觉表现：只根据额度健康度给主模块中的大数字上色。这个改动用于帮助用户快速判断当前 Primary Provider 的额度是否充足、接近预警、或已经告急；不改变任何 provider 数据、菜单栏标题、状态文案、二级供应商列表或 Console UI。

## 目标

- 给详情页主模块里的余额、已用 plan、剩余额度大数字上色。
- 良好状态使用现有“额度可用”状态文案的绿色作为参考。
- 一般状态使用黄色，告急状态使用红色。
- 保持菜单栏标题、状态文案颜色、状态点颜色、二级供应商列表、Console 页面不变。
- 在 Core display model 层用自动化测试覆盖阈值边界。

## 非目标

- 不修改菜单栏标题或菜单栏图标文字。
- 不修改 provider 状态文案、状态 tone 或状态点行为。
- 不给二级供应商列表数字上色。
- 不修改 Console Home、Console API 或其他 Console 数字展示。
- 不增加通知、弹窗、预算预测或历史图表。
- 不修改 provider 拉取逻辑、API 解析、刷新频率、凭据或持久化。

## 颜色规则

颜色规则只作用于菜单栏展开详情页主模块中的大数字。

### 余额类 Provider

DeepSeek 这类充值余额 provider 使用总余额金额判断。

- `totalBalance >= 50`：良好，显示绿色。
- `totalBalance >= 10` 且 `< 50`：一般，显示黄色。
- `totalBalance < 10`：告急，显示红色。

边界值统一归入更健康的一档。例如 `50` 是绿色，`10` 是黄色。

### Used Plan 类 Provider

智谱 GLM Coding Plan 这类 provider 使用已用百分比判断。

- `usagePercentage <= 40`：良好，显示绿色。
- `usagePercentage > 40` 且 `<= 80`：一般，显示黄色。
- `usagePercentage > 80`：告急，显示红色。

边界值统一归入更健康的一档。例如 `40` 是绿色，`80` 是黄色。

### Remaining Quota 类 Provider

OpenAI/Codex 这类 provider 使用每个展示窗口的剩余额度百分比判断。

- `remainingPercentage >= 60`：良好，显示绿色。
- `remainingPercentage >= 20` 且 `< 60`：一般，显示黄色。
- `remainingPercentage < 20`：告急，显示红色。

边界值统一归入更健康的一档。例如 `60` 是绿色，`20` 是黄色。

## 架构

在 Core 层增加一个主模块数字 tone 类型，包含良好、一般、告急，以及用于占位值的 neutral fallback。阈值逻辑属于 provider 额度语义，不属于 SwiftUI 样式，因此放在 Core display model 中。

扩展 `PrimaryProviderDisplayParts`，为余额和 used-plan 主模块的大数字携带 amount tone。扩展 `QuotaWindowDisplayRow`，为 OpenAI/Codex 主模块的每个额度窗口携带 amount tone，因为不同窗口可能有不同剩余额度，也可能对应不同颜色。

`MenuBarContentView` 只负责把 Core tone 映射成 SwiftUI 颜色：

- 良好 -> `.green`
- 一般 -> `.yellow`
- 告急 -> `.red`
- neutral -> `.primary`

视图只在以下位置应用颜色：

- `balanceAmountLine(parts:amountSize:)` 中的 `Text(parts.amountText)`
- `quotaHeroRows` 中的 `Text(row.amountText)`

其他文字保持现有 foreground style。

## 数据流

1. Provider 继续返回现有 `ProviderSnapshot`。
2. `ProviderDisplayFormatter.primaryDisplayParts(...)` 为余额和 used-plan snapshot 计算 amount tone。
3. `MenuBarBalanceViewModel.primaryQuotaWindowRows` 通过 Core formatter/helper 为每个 remaining-quota window 计算 amount tone。
4. `MenuBarContentView` 只用对应颜色渲染主模块数字文本。

## 错误处理和兜底

`--`、缺少 snapshot、没有 last snapshot 的 loading 状态、未配置状态都使用 neutral tone，并以 `.primary` 渲染。失败状态如果保留了 last snapshot，则主模块数字继续使用该 last snapshot 对应的 amount tone。

## 测试

Core 测试直接验证阈值逻辑，不需要检查 SwiftUI 内部结构：

- DeepSeek 余额边界：`50`、`49.99`、`10`、`9.99`。
- 智谱 used-plan 边界：`40`、`40.01`、`80`、`80.01`。
- OpenAI/Codex remaining-quota 边界：`60`、`59`、`20`、`19`。
- Primary display parts 对余额和 used-plan snapshot 暴露正确 tone。
- Quota window display rows 对每个窗口暴露正确 tone。
- 二级供应商列表不扩展也不渲染主模块数字 tone。

手动验证时，在测试通过后运行本地菜单栏应用，确认只有展开详情页主模块的大数字发生颜色变化。
