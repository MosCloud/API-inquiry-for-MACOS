# API Inquiry v0.3.6 API 页面 UI 设计

## 摘要

v0.3.6 优化 Console 的 API 页面，把当前高卡片、按钮同权重的布局改为紧凑的供应商接入管理模块。每个供应商仍然是独立卡片，左侧沿用 Home 页中的供应商入口视觉，右侧聚焦后续管理操作。Home 继续承担添加供应商和首次接入入口，API 页面承担后续密钥、配置和供应商管理。

## 目标

- 保留每个供应商独立卡片模块，而不是合并成表格或大列表容器。
- 卡片高度显著低于 Home 供应商行，采用单行主信息结构。
- 左侧供应商显示沿用 Home 页现有 provider chip：Logo/名称/外部跳转箭头。
- 中间状态区位于中列，内部左对齐，不靠右贴近操作区。
- DeepSeek 显示 `已配置` 和 `可用于充值余额查询`。
- 智谱 BigModel 显示 `已配置` 和 `可用于套餐余额查询`。
- OpenAI/Codex 显示 `已加载` 和 `可用于套餐余额查询`。
- 删除 API 页中关于 Keychain 的解释性说明。
- DeepSeek/智谱主按钮用于配置或更换 API 密钥。
- OpenAI/Codex 主按钮为 `打开配置`，打开本机 Codex `auth.json` 或其配置目录。
- 删除密钥、移除供应商等危险/低频操作收进更多菜单并保留确认。
- 删除密钥确认文案不再提及 Keychain/密钥串，只描述“删除已保存的 API 密钥”。

## 非目标

- 不改变 Home 页供应商概览、健康度染色、添加供应商主入口。
- 不改变供应商刷新频率、余额/套餐查询协议或凭据存储策略。
- 不重新设计导航栏、Settings 页或菜单栏详情页。
- 不在 API 页重复说明安全、隐私、Keychain 等下载说明页面已有内容。
- 不暴露任何 API Key 或 Codex token。

## 页面分工

Home 页是发现、添加和首次接入入口。用户从 Home 添加供应商后，可以进入 API 页完成首次密钥录入。API 页是后续接入管理页面，负责查看接入状态、配置/更换密钥、删除密钥、移除供应商，以及打开 Codex 本机配置。

## API 卡片布局

每个供应商卡片使用三段式单行布局：

```text
[供应商 chip ↗]      [状态] 用途说明                 [主按钮] [...]
```

- 左列：复用 Home 页供应商 chip 样式和外部跳转行为。
- 中列：状态 pill + 用途文案，整体在中列左对齐。
- 右列：主操作按钮和更多菜单，右对齐。

推荐卡片高度为 76-88 pt，卡片间距 10-12 pt。背景、圆角和半透明层级延续 Home 页的 8 pt continuous 圆角与安静材质感。

## 状态与操作

| 供应商类型 | 状态 | 用途文案 | 主操作 |
| --- | --- | --- | --- |
| 充值余额 API，例如 DeepSeek | 已配置 / 未配置 | 可用于充值余额查询 | 配置密钥 / 更换密钥 |
| 套餐余额 API，App 管理密钥，例如智谱 | 已配置 / 未配置 | 可用于套餐余额查询 | 配置密钥 / 更换密钥 |
| 套餐余额 API，本机配置加载，例如 OpenAI/Codex | 已加载 / 未加载 | 可用于套餐余额查询 | 打开配置 |

更多菜单：

- 支持 App 管理密钥的供应商：删除 API 密钥、移除供应商。
- OpenAI/Codex：在 Finder 中显示配置、移除供应商。

外部供应商页面由左侧 chip 的斜箭头承担，不再在主按钮或更多菜单中重复。

## OpenAI/Codex 配置

OpenAI/Codex provider 继续使用现有 `CodexCredentialStore` 从本机配置读取凭据。API 页需要获得一个可打开的位置：

- 如果可用的 `auth.json` 存在，`打开配置` 打开该文件。
- 如果 `auth.json` 不存在，打开首选配置目录，例如 `$CODEX_HOME` 或 `~/.codex`。
- 如果无法解析配置目录，给出非敏感反馈，不显示 token。

配置路径判断属于 Core/安全层或 ViewModel 的 display model，不应在 SwiftUI 中复制环境变量逻辑。

## 交互状态

- 未配置的 API Key 管理型供应商在卡片内展开输入态，保存/取消按钮与当前流程一致。
- 替换密钥时只显示新密钥输入框，不显示旧密钥。
- 删除密钥和移除供应商继续使用二次确认，确认 UI 可以在卡片下方紧凑展开。
- 删除密钥确认只说明用户动作，不重复底层存储实现。
- 供应商数量为 1 时继续禁用移除供应商。

## 测试

- Core 测试覆盖 API 页面 status/use-case 文案：DeepSeek 充值余额、智谱套餐余额、Codex 已加载/未加载。
- Core 测试覆盖 Codex 配置 URL 选择：存在 auth 文件时返回文件，不存在时返回配置目录。
- 本地化测试覆盖删除 API Key 确认文案不包含 Keychain/密钥串。
- Swift build 验证 SwiftUI 编译。
- 手动视觉验证 API 页面保持独立紧凑卡片、左侧 chip 与 Home 一致、中列左对齐、右侧按钮不挤压文本。
