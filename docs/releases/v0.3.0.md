# API Inquiry v0.3.0

## New Features

- Added a built-in multi-provider catalog with DeepSeek and Zhipu GLM Coding Plan as the first providers.
- Added Zhipu GLM Coding Plan usage checks, including plan window and usage percentage display.
- Added plan reset visibility for Zhipu: `Resets` in the menu details panel and `Plan Next Resets` in Console Home.
- Changed the menu bar to show the Primary Provider: DeepSeek shows balance, while Zhipu shows plan usage such as `5h 17%`.
- Updated the menu bar details panel to feature the Primary Provider at the top and show other added providers as compact rows.
- Added Console management for adding providers, saving/replacing/deleting provider-specific API keys, and selecting the Primary Provider shown in the menu bar.
- Kept each provider isolated with its own Keychain account, refresh state, last snapshot, and error state.

## Improvements

- Refined the Home and API layouts so provider cards, metric columns, actions, and provider logos align consistently.
- Updated the menu details refresh action to refresh every added provider, not only the Primary Provider.
- Polished DeepSeek and Zhipu detail formatting, including currency typography, plan usage typography, and system 12-hour/24-hour time formatting.

## Known Limits

- No Apple notarization.
- Only built-in providers are supported; arbitrary custom endpoints are not supported.
- Historical usage import, charts, and cross-provider aggregate statistics are not included.
