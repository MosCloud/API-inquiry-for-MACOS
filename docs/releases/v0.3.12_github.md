# API Inquiry v0.3.12

## 中文

- 修复 OpenAI/Codex 额度窗口识别：当上游仅返回 1 周额度时，应用会正确显示为周额度，不再误标为 5 小时额度。
- 菜单栏额度缩写统一为 `5H` 和 `1W`，便于快速辨识当前额度窗口。

### 应用优化

- 更新应用与打包版本号为 v0.3.12。

### 下载校验

```bash
shasum -a 256 -c API-Inquiry-v0.3.12.dmg.sha256
```

### 已知限制

- Apple notarization 尚未启用。
- 手动重置额度依赖当前本机 Codex/ChatGPT 凭证；如果凭证过期，需要先刷新登录态再查询。

## English

### Bug Fixes

- Fixed OpenAI/Codex quota-window recognition: when the upstream service returns only a weekly quota, the app now identifies it as weekly instead of mislabeling it as a five-hour quota.
- Standardized compact menu-bar quota labels as `5H` and `1W` for quicker window recognition.

### App Optimization

- Updated the app and packaging version to v0.3.12.

### Verify Downloads

```bash
shasum -a 256 -c API-Inquiry-v0.3.12.dmg.sha256
```

### Known Limitations

- Apple notarization is not enabled.
- Manual reset credits depend on the current local Codex/ChatGPT credentials. If those credentials expire, refresh the login before querying again.
