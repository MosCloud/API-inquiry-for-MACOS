# API Inquiry v0.3.6

## New Features

- Added an OpenAI/Codex "Open Config" action on the API page so users can quickly locate the local auth configuration used for plan quota checks.

## App Optimization

- Redesigned the Console API provider management area into compact standalone cards, aligning provider identity, access status, purpose text, and management actions in one row.
- Reused the Home page provider chip treatment on the API page so DeepSeek, Zhipu BigModel, and OpenAI have a more consistent brand presentation.
- Tightened provider card heights and spacing on both Home and API pages for a denser macOS Settings-style management surface.
- Tuned DeepSeek, Zhipu BigModel, and OpenAI logo sizing inside provider chips so the three providers have better visual balance.
- Refined API management button copy and sizing, shortening the key action to "Replace Key" and making "Replace Key" and "Open Config" easier to scan and click.
- Clarified API access purpose text: balance providers now describe recharge balance checks, while plan providers describe plan quota checks.
- Bumped app metadata and release packaging configuration to v0.3.6 so Settings shows v0.3.6.

## Bug Fixes

- Fixed OpenAI/Codex plan quota parsing when the API returns null reset fields, which could otherwise make the Home page show OpenAI as unavailable.
- Fixed unavailable providers on the Home page using warning-colored status badges for authentication, network, parsing, or unknown failures; these states now use neutral gray styling.
