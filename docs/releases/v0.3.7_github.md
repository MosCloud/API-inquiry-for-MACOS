# API Inquiry v0.3.7

## 中文

### 应用优化

- 增加克制的刷新、错误、状态和设置反馈微交互，保持菜单栏与 Console 继续为内容服务。
- 将刷新反馈改为确定性的 0.8s 前进式旋转循环：真实刷新状态结束后自然停止，并尊重 Reduce Motion。
- 为设置/API 结果反馈增加轻量状态图标，让成功、警告和错误更容易扫读，不引入 toast 或模态弹层。
- 将 warning 色统一为更克制的共享 amber 表达，兼顾浅色和深色外观。
- 补充菜单栏状态、额度行、供应商行、状态 badge、更多操作按钮、刷新、Console、开机自启和退出控件的可访问性 label/value。
- 增加 `More Actions`、`Current Status`、`Quota Window` 对应的中英文辅助文案。
- 新增 App target 集中的高版本 SwiftUI 效果 helper，并保留 macOS 13 fallback。
- 从 Core display model 暴露真实数值，使 macOS 14+ `numericText(value:)` 能用于余额、用量百分比和额度百分比变化，不解析格式化后的展示字符串。
- 仅对明确的设置/结果反馈启用 macOS 14+ sensory feedback。

### 问题修复

- 修复 API access badge 语义：状态颜色和文案改由显式 access state 驱动，不再依赖本地化字符串比较。
- 系统开启 Reduce Motion 时，本轮新增动效会降级或关闭。
- 保留 Console Home 供应商行内部按钮的独立无障碍访问，不再把整行压平成不完整的合并文案。

### 下载校验

```bash
shasum -a 256 -c API-Inquiry-v0.3.7.dmg.sha256
```

### 已知限制

- Apple notarization 尚未启用。
- 本版本不新增供应商，也不进入 v0.4.0 通用 Provider 能力开发。

## English

### App Optimization

- Added restrained refresh, error, state, and settings feedback microinteractions while keeping the menu bar and Console content-first.
- Reworked the refresh affordance into a deterministic 0.8s forward-only rotation loop that stops when the real refresh state ends and respects Reduce Motion.
- Added concise feedback icons for settings/API result messages so success, warning, and error states are easier to scan without adding toasts or modal UI.
- Updated warning tone usage to a quieter shared amber treatment for dark and light appearances.
- Added accessibility labels and values for menu status, quota rows, provider rows, status badges, more-action buttons, refresh, Console, launch-at-login, and quit controls.
- Added localized helper copy for "More Actions", "Current Status", and "Quota Window".
- Added a centralized App-target helper for higher-version SwiftUI effects with macOS 13 fallback.
- Exposed real numeric display values from Core models so macOS 14+ `numericText(value:)` can animate balances, usage percentages, and quota percentages without parsing formatted strings.
- Added macOS 14+ sensory feedback only to concrete settings/result feedback.

### Bug Fixes

- Repaired API access badge semantics so status color and copy are driven by explicit access state instead of localized string comparison.
- Reduced newly added motion when the system Reduce Motion setting is enabled.
- Preserved individual Console Home row controls for assistive technologies instead of flattening each provider row into an incomplete combined label.
