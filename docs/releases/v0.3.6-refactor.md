# API Inquiry v0.3.6-Refactor

This release makes API Inquiry feel steadier and easier to use while preparing the app for more providers in future versions. You do not need to learn a new workflow: the menu bar, Console, API key setup, and language switching all work the same way as before.

## What You Will Notice

- Multi-provider refresh feels smoother. DeepSeek, Zhipu, and OpenAI/Codex refresh independently, so one slow or failing provider does not block the rest of the status view.
- The Console is easier to scan. Home, API, and Settings are organized more clearly, making provider status, balance, quota, reset time, and key status easier to understand at a glance.
- The menu bar display is more reliable. The provider shown in the menu bar is handled more consistently, which keeps the experience predictable as the app grows.
- API key handling is safer. Keys still stay local on your Mac, and Codex configuration is read in a more careful, read-only way whenever possible.
- Settings now shows `v0.3.6-Refactor`, so you can confirm you are running this refreshed build.

## Why This Update Matters

This is not a flashy feature release. It is a cleanup release that makes the current app more dependable and gives future provider support a better foundation.

In short: the app should feel familiar today, and easier to improve tomorrow.

## Upgrade Notes

- You can install this version over the previous build.
- Existing API keys and local settings continue to work.
- Apple notarization is still not enabled. If macOS warns about the app, follow the install instructions in the README.
