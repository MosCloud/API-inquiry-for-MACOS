# API Inquiry v0.3.6-Refactor

## App Optimization

- Refreshed this refactor release with a follow-up provider runtime cleanup after UI validation.
- Centralized built-in provider metadata into descriptors and registrations, so provider display names, menu prefixes, credential accounts, access purpose, credential policy, and homepages now come from one source.
- Made `ProviderRegistration` the runtime source of provider descriptors: `MultiProviderBalanceCoordinator` now builds runtimes from registrations instead of falling back through the global catalog or provider metadata.
- Hardened provider runtime defaults: custom registration sets can omit `defaultProviderID`, the coordinator now derives it from the first registration, and invalid explicit defaults are rejected before provider factories are invoked.
- Kept `BalanceProvider` focused on API fetching by removing descriptor/display/credential convenience metadata from the protocol extension.
- Clarified that provider factories should stay lightweight and avoid credential, network, or persistent storage side effects.
- Injected descriptor-owned credential accounts into `BalanceRefreshController`, so refresh logic no longer asks providers for Keychain account names.
- Unified menu bar and Console ViewModels around `MultiProviderBalanceCoordinator`, removing the legacy single-provider routing path from production code.
- Split the oversized Console UI into focused SwiftUI views for navigation, section headers, Home provider rows, API provider panels, Settings, badges, logos, metrics, and design tokens.
- Split provider display formatting into value formatting, status copy, tone resolution, and display models while keeping `ProviderDisplayFormatter` as a compatibility facade.
- Removed provider-based formatter overloads and simplified provider mocks so tests no longer imply that metadata lives on provider instances.
- Moved provider image/logo handling from the DeepSeek-specific helper into `ProviderVisualCatalog`.
- Updated Settings to show the refactor build label `v0.3.6-Refactor`.

## Behavior Improvements

- Multi-provider refresh now runs providers concurrently while each provider keeps its own loading/error state isolation.
- Codex configuration target lookup is now read-only by default; directory creation is available through an explicitly named API.
- Console credential-management actions now fail closed when provider descriptor metadata is unexpectedly unavailable.
