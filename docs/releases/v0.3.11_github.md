# API Inquiry v0.3.11

## 中文

### 应用优化

- 新增 GitHub Actions 基线，会构建 Swift package 并运行本地测试 runner。
- 额度窗口处理改用稳定窗口语义，不再依赖显示标签。
- 改进 Console 在浅色模式下的外观一致性。
- 统一 Console 反馈处理，避免供应商消息叠加。
- 菜单刷新和手动重置刷新控件共享刷新反馈状态。

### 问题修复

- 修复 Codex 手动重置缓存跟踪，使其不再依赖进程随机化的字符串哈希。
- 当本机 Codex `auth.json` 存在但格式异常或缺少 access token 时，新增可见警告。
- 修复 Console 窗口尺寸与 SwiftUI 内容尺寸不一致的问题。

### 下载校验

```bash
shasum -a 256 -c API-Inquiry-v0.3.11.dmg.sha256
```

### 已知限制

- Apple notarization 尚未启用。
- 手动重置额度依赖当前本机 Codex/ChatGPT 凭证；如果凭证过期，需要先刷新登录态再查询。

## English

### App Optimization

- Added a GitHub Actions baseline that builds the Swift package and runs the local test runner.
- Made quota-window handling use stable window semantics instead of relying on display labels.
- Improved Console appearance consistency in light mode.
- Unified Console feedback handling so provider messages do not stack.
- Shared refresh feedback state between menu refresh and manual-reset refresh controls.

### Bug Fixes

- Fixed Codex manual-reset cache tracking so it no longer depends on process-randomized string hashes.
- Added a visible warning when a local Codex `auth.json` exists but is malformed or missing an access token.
- Fixed the Console window size mismatch between the SwiftUI content and AppKit window.

### Verify Downloads

```bash
shasum -a 256 -c API-Inquiry-v0.3.11.dmg.sha256
```

### Known Limitations

- Apple notarization is not enabled.
- Manual reset credits depend on the current local Codex/ChatGPT credentials. If those credentials expire, refresh the login before querying again.
