# API Inquiry v0.3.6-Refactor

## 中文

### 应用优化

- 将内置 Provider 元数据集中到 descriptor 和 registration，供应商显示名、菜单栏前缀、凭据账户、接入用途、凭据策略和主页现在都从同一来源派生。
- 将菜单栏和 Console ViewModel 统一到 `MultiProviderBalanceCoordinator`，移除历史 single-provider 路由路径，同时保留用于测试和少量调用方的兼容初始化入口。
- 拆分过大的 Console UI 文件，将导航、section header、Home 供应商行、API 供应商面板、Settings、badge、logo、指标格和设计 token 拆成独立 SwiftUI View。
- 将 provider 显示格式拆分为数值格式、状态文案、tone 判断和显示模型，同时保留 `ProviderDisplayFormatter` 作为兼容门面。
- 将供应商图片/logo 逻辑从 DeepSeek 专属 helper 收敛到 `ProviderVisualCatalog`。
- Settings 页面版本号显示为本次重构构建标签 `v0.3.6-Refactor`。

### 行为改进

- 多 Provider 刷新现在并发执行，同时每个 Provider 继续保持独立的 loading/error 状态隔离。
- Codex 配置目标查询默认变为只读；需要创建目录的路径改为显式命名 API。

### 下载校验

```bash
shasum -a 256 -c API-Inquiry-v0.3.6-Refactor.dmg.sha256
```

### 已知限制

- Apple notarization 尚未启用。
- 这是为下一阶段 Provider 扩展能力做验证的重构版本。

## English

### App Optimization

- Centralized built-in provider metadata into descriptors and registrations, so provider display names, menu prefixes, credential accounts, access purpose, credential policy, and homepages now come from one source.
- Unified menu bar and Console ViewModels around `MultiProviderBalanceCoordinator`, removing the legacy single-provider routing path while preserving compatibility initializers for tests and narrow callers.
- Split the oversized Console UI into focused SwiftUI views for navigation, section headers, Home provider rows, API provider panels, Settings, badges, logos, metrics, and design tokens.
- Split provider display formatting into value formatting, status copy, tone resolution, and display models while keeping `ProviderDisplayFormatter` as a compatibility facade.
- Moved provider image/logo handling from the DeepSeek-specific helper into `ProviderVisualCatalog`.
- Updated Settings to show the refactor build label `v0.3.6-Refactor`.

### Behavior Improvements

- Multi-provider refresh now runs providers concurrently while each provider keeps its own loading/error state isolation.
- Codex configuration target lookup is now read-only by default; directory creation is available through an explicitly named API.

### Verify Downloads

```bash
shasum -a 256 -c API-Inquiry-v0.3.6-Refactor.dmg.sha256
```

### Known Limitations

- Apple notarization is not enabled.
- This is a refactor release for validation before the next provider-expansion version.
