# API Inquiry v0.3.2

## New Features

- Added Chinese UI. When the Mac's preferred language is Chinese, API Inquiry shows Chinese by default.
- Added a language setting in Console Home: `Auto / 中文 / English`.
- The language selection is persisted. The default `Auto` mode follows the system language.
- The menu details panel and Console update together after language changes.
- The menu bar status text now uses the official macOS menu bar font, size, and weight.

## Localization Coverage

- Menu details, Console Home, Console API, buttons, statuses, error messages, settings feedback, last-updated text, and reset text support both Chinese and English.
- `API Key` is localized as `API 密钥` in Chinese.
- `Keychain` is localized as `密钥串` in Chinese.
- Chinese UI uses compact Codex quota labels in details: `5 时` and `1 周`.
- Chinese UI keeps menu bar quota text as `5h xx%`, matching English and reducing status bar width changes.
- Console Home uses `更新于` for provider updated-time metrics.

## Kept As-Is

- Provider and product names stay unchanged, including `OpenAI`, `DeepSeek`, `Zhipu GLM Coding Plan`, `Codex`, and `ChatGPT`.
- Common symbols and plan names stay unchanged, including `API`, `CNY`, `Free`, `Plus`, `Pro`, `Prolite`, and `Business`.

## Known Limitations

- Apple notarization is not enabled.
- Only Chinese and English are supported.
- Language switching does not change provider query logic, saved API keys, Codex login state, Primary Provider, or last snapshots.
