# API Inquiry v0.3.1

## New Features

- Added an OpenAI/Codex provider that queries Codex quota through the ChatGPT/Codex session quota endpoint.
- OpenAI/Codex first auto-reads local Codex login state (`$CODEX_HOME/auth.json` or `~/.codex/auth.json`), so users do not need to enter an OpenAI Platform API key in the Console.
- When OpenAI/Codex is the Primary Provider, the menu bar keeps the compact image-label design and shows the OpenAI icon with `5h xx%` remaining quota.
- The OpenAI/Codex detail panel shows both `5h` and `7d` remaining quota windows; `5h` resets show time, while `7d` resets show date.
- When OpenAI/Codex is a secondary provider, it is shown as OpenAI and displays `5h xx%` and `7d xx%` on separate lines.
- Console Home shows the current OpenAI/Codex plan, such as Free, Plus, Pro 5x, Pro 20x, Business, or Enterprise/Edu.
- Console Home Detail labels quota semantics explicitly: Zhipu shows `used`, and OpenAI/Codex shows `remg`.

## UI Polish

- Added OpenAI header/logo assets and a 16x16 menu icon for consistent OpenAI branding in the detail panel and Console.
- Removed secondary-provider `Last updated` rows from the detail panel, leaving the timestamp only on the primary display unit.
- When Zhipu is a secondary provider, `5h xx%` and `Resets: xx:xx` are shown on one line.

## Known Limits

- No Apple notarization.
- OpenAI/Codex quota checks depend on a ChatGPT/Codex session backend that may change with Codex itself.
- Only built-in providers are supported; arbitrary custom endpoints are not supported.
