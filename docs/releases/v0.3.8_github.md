# API Inquiry v0.3.8

## 中文

### 问题修复

- 修复 OpenAI/Codex 额度响应不完整时，界面可能短暂只显示 5 小时窗口并隐藏 7 天窗口的问题。
- API Inquiry 现在会对不完整的 OpenAI/Codex 额度响应自动重试一次，再判断本次刷新不可用。
- 如果重试后 OpenAI/Codex 响应仍不完整，应用会保留上一次完整额度快照，不再用残缺数据覆盖界面。
- OpenAI/Codex 额度百分比现在会保留接口返回的小数精度，与官方 Codex 显示更一致。

### 下载校验

```bash
shasum -a 256 -c API-Inquiry-v0.3.8.dmg.sha256
```

### 已知限制

- Apple notarization 尚未启用。
- 本版本聚焦 OpenAI/Codex 额度刷新稳定性，不新增供应商。

## English

### Bug Fixes

- Fixed an issue where OpenAI/Codex quota could briefly show only the 5-hour window and hide the 7-day window when the usage response was incomplete.
- API Inquiry now retries incomplete OpenAI/Codex quota responses once before treating the refresh as unavailable.
- If an incomplete OpenAI/Codex response still cannot be recovered, the app keeps the last complete quota snapshot instead of replacing it with partial data.
- OpenAI/Codex quota percentages now preserve fractional values from the service response, improving consistency with the official Codex display.

### Verify Downloads

```bash
shasum -a 256 -c API-Inquiry-v0.3.8.dmg.sha256
```

### Known Limitations

- Apple notarization is not enabled.
- This release focuses on OpenAI/Codex quota refresh stability and does not add new providers.
